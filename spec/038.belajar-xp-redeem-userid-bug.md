## 2. Tujuan yang Dipahami

Memperbaiki **dua buah bug** di halaman Belajar:
1. **XP Hilang / Reset:** Setelah kerjakan soal $\rightarrow$ pindah menu (Beranda) $\rightarrow$ kembali ke Belajar $\rightarrow$ XP hilang dan quiz kembali berstatus belum dikerjakan.
2. **Gagal Redeem:** Saat klik tombol Redeem (Tukar 40 XP $\rightarrow$ \$5,000), muncul error "User tidak ditemukan".

---

## 3. Codebase Analysis (Penyebab Bug)

Kedua bug ini ternyata memiliki akar masalah yang *sama persis* dengan bug "userId dan materiId wajib diisi" pada spec sebelumnya (036). 

Akar masalahnya ada pada cara frontend membaca ID user dari `localStorage` (dimana response backend menyimpan ID di property `id`, tetapi frontend mencoba mengakses `_id`).

### 3.1 Kenapa XP hilang saat pindah menu?

Ketika user pindah dari menu "Belajar" ke menu "Beranda", komponen `<Belajar />` di-unmount dari React DOM (state hancur/hilang). Saat user kembali ke menu "Belajar", komponen dimuat ulang (*mount*) dan menjalankan instruksi awal (`useEffect`) untuk menarik data:

```typescript
// Belajar.tsx baris 123-124
if (user?._id) { 
  fetch(`http://localhost:5000/api/nilai/user/${user._id}`)
  // ...
}
```

Karena `user._id` adalah `undefined` (hanya ada `user.id`), kondisi statemen `if` menjadi **FALSE**. Akibatnya, API `GET /api/nilai/user/...` **tidak pernah dipanggil!** 

Karena data XP dari backend tidak ditarik, state `nilaiMap` tetap kosong. Sehingga UI menampilkan seolah-olah user belum pernah mengerjakan soal dan XP-nya `0`.

### 3.2 Kenapa Redeem error "User tidak ditemukan"?

Masalah serupa terjadi di fungsi `handleRedeem` (Belajar.tsx baris 169):

```typescript
// Belajar.tsx baris 169
body: JSON.stringify({ userId: user._id, amountXp: amount }),
```

Karena `user._id` adalah `undefined`, payload yang dikirim ke backend menjadi `{ userId: undefined, amountXp: 40 }`.

Di sisi backend (`nilaiController.js` baris 106-107):
```javascript
const user = await User.findById(userId); // <= findById(undefined) akan null
if (!user) return res.status(404).json({ success: false, message: "User tidak ditemukan" });
```
Karena `userId` undefined, backend gagal menemukan user di database dan memberikan pesan error *"User tidak ditemukan"*.

## 5. Logic / Pseudocode

`useEffect`:
```
ON COMPONENT MOUNT:
  Get user from localStorage
  Set userId = user._id OR user.id
  IF userId exists:
    Fetch "/api/nilai/user/{userId}"
    Populate nilaiMap, totalEarnedXp, redeemedXp
    UI re-renders displaying true persistence
```

`handleRedeem`:
```
BUTTON CLICKED:
  Get user from localStorage
  Set userId = user._id OR user.id
  Fetch "/api/nilai/redeem", POST { userId, amountXp: 40 }
  Backend subtracts XP, adds Balance ($5,000)
  Return success
```

## 6. Tracing Scenarios

1. **User kembali dari "Beranda" ke "Belajar"**:
   - `Belajar` mount.
   - Baca localStorage $\rightarrow$ `currentUserId = "abc"`.
   - `fetch("/api/nilai/user/abc")` berjalan!
   - Response dari backend mengembalikan *record* quiz yang sudah dikerjakan beserta `redeemedXp`.
   - State `nilaiMap` terisi $\rightarrow$ Quiz yang sudah selesai masuk ke daftar "selesai" di UI.
   - XP 40 kembali terpampang nyata di layar ✅.

2. **User menekan "Tukar 40 XP $\rightarrow$ $5,000"**:
   - `handleRedeem` berjalan.
   - `userId = "abc"`.
   - Payload POST $\rightarrow$ `{ "userId": "abc", "amountXp": 40 }`.
   - Backend memproses $\rightarrow$ Kurangi 40 XP, tambah $5000 ke balance user.
   - Response OK $\rightarrow$ Toast hijau "Berhasil" ✅.
