## 2. Tujuan yang Dipahami
Memperbaiki bug kalkulasi Keuntungan (Profit/Loss) pada tab **Riwayat Transaksi** di menu Simulasi. 
Kondisi saat ini, profit dari setiap baris riwayat transaksi (misal transaksi Ethereum atau Cardano terdahulu) dihitung menggunakan harga live koin yang sedang **aktif terpilih (selectedCoin)** di dropdown paling atas (misal Bitcoin). Hal ini menyebabkan nilai profit menjadi *terlalu optimis* (angka % melonjak tidak masuk akal jika harga koin pilihan saat ini jauh lebih tinggi dari koin transaksi), atau *terlalu pesimis* (jika harganya lebih kecil).

**Tujuan Implementatif:**
Mengisolasi harga per baris transaksi riwayat agar di-evaluasi terhadap *live price* khusus koin itu sendiri, bukan merujuk pada `selectedCoin`. Walaupun pengguna sedang memonitor menu dropdown untuk Bitcoin, baris riwayat ETH tetap akan dinilai dengan harga live ETH dari list harga pasar saat itu, demikian halnya dengan Cardano.

## 4. Perubahan Secara Logika & Code

### A. Modifikasi Type Definiton
Mengekspansi tipe struktur balikan data dari state `history`.
```tsx
type Transaction = {
  type: "BUY" | "SELL";
  coinId?: string; // Menampung raw coin_id (ex: 'ethereum')
  name: string;
  price: number;
  quantity: number;
  total: number;
};
```

### B. Patching Algoritma di perulangan Map Riwayat
Melakukan lokalisasi nilai `currentPrice` dengan menelusuri array `coins` ter-update yang diambil di awal halaman load untuk menemukan detail harga market-live *pada koin tersebut saja*.

**Code Perubahan di dalam `history.map((tx, i) => {...})`:**
```tsx
{history.map((tx, i) => {
  // FIND CURRENT LIVE PRICE OF THE *SPECIFIC* TRANSACTION COIN, NOT SELECTEDCOIN
  const txCoinMatch = coins.find((c) => c.id === tx.coinId || c.name === tx.name);
  
  // Ambil harganya. Jika karena suatu alasan coin tak ditemukan dari live list, fallback rate profit ke 0 / default dengan ambil currentPrice = tx.price
  const currentPrice = txCoinMatch ? txCoinMatch.current_price : tx.price; 

  const profit =
    tx.type === "BUY"
      ? (currentPrice - tx.price) * tx.quantity
      : (tx.price - currentPrice) * tx.quantity;
  
  const percent =
     tx.price > 0
        ? (profit / (tx.price * tx.quantity)) * 100
        : 0;
  
  const isProfit = profit >= 0;
  
  return (
    // Render HTML card history item...
  )
})}
```

### Mengapa Perubahan Ini Dilakukan?
Menyingkirkan dependensi tunggal (`selectedCoin`) pada loop merender array majemuk (Multi-Coin History). Dengan mencocokkan `tx.coinId` terhadap database array list `coins` (harga realtime 15+ coins dari backend), maka presentase % riwayat simulasi pada Dashboard murni akurat mengikuti harga bursa per-koin aslinya.

## 5. Tracing / Scenario Testcase

**Skenario Simulasi Bug vs Fix:**

- **Harga Dasar:** 
  - `BTC` Live: $65,000
  - `ETH` Live: $3,500
  - `ADA` Live: $0.45
- **Aksi User:** User A membeli 2 qty ETH di harga konstan $3,500. Total Cost $7,000.
- **Transisi Bug:**
  User A kemudian mengganti Dropdown Koin pilihannya di atas layar ke **BTC**.
  *(Kondisi Sebelum Fix)* -> Loop history merender currentPrice menjadi $65,000. Profit ETH yang dideklarasi: `(65,000 - 3,500) * 2` = Profit meledak menjadi $123,000 (+1,700%). Sangat optimis karena dicalc dengan price BTC.
- **Transisi Fix:**
  *(Kondisi Sesudah Fix)* -> Loop history melakukan validasi `txCoinMatch` menggunakan ID tx history "ethereum". Ditemukan objek ETH di array live `coins` berupa $3,500. `currentPrice` disetel ke $3,500.
  Profit ETH dideklarasi: `(3,500 - 3,500) * 2` = Profit Netral $0 (+0%). Persentase akurat terhadap mata uang itu sendiri terlepas dari dropdown apa yang sedang tersentuh.
