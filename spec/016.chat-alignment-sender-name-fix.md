## 2. Tujuan yang Dipahami

Memperbaiki **2 Bug Utama** di halaman Chat (`Chat.tsx`) yang merusak pengalaman WhatsApp-style:
1. **Message Alignment Error:** Semua pesan real-time (baik dari kita maupun lawan bicara) muncul di sebelah **kanan** (seolah-olah semuanya dikirim oleh kita).
2. **Sender Name Mismatch:** Saat user chat dengan mentor, nama di header chat malah menampilkan "User" (nama diri sendiri) atau nama yang salah, bukan nama mentor.

---

## 3. Codebase Analysis & Root Cause

### 3.1 Kenapa Header Menampilkan "User"?

Di `Chat.tsx` (baris 126-138), untuk mendapatkan nama partner chat (lawan bicara), kode memfilter participants:

```typescript
const partner = conv.participants.find(
  (p: any) => p._id !== user._id
);
```

**Masalah:** Objek `user` yang diambil dari `localStorage` (**hasil login dari `auth.js`**) berformat seperti ini:
```json
{
  "id": "65f0123...",
  "name": "Nama User",
  "role": "user"
}
```
Perhatikan bahwa key-nya adalah **`id`**, bukan **`_id`**.
Karena `user._id` adalah `undefined`, kondisi `p._id !== undefined` selalu **true** untuk semua participant (karena `p._id` berupa string).
Akibatnya, `.find()` selalu mengembalikan participant pertama (yaitu diri kita sendiri). Itulah sebabnya nama di header selalu nama diri kita sendiri ("User").

### 3.2 Kenapa Semua Pesan Muncul di Kanan?

Di `Chat.tsx` (baris 311-313), penentuan apakah sebuah pesan adalah milik kita (`isMe`) menggunakan logika ini:

```typescript
const isMe =
  msg.sender._id === user._id ||
  (msg.sender as any) === user._id;
```

**Skenario saat pesan baru dikirim/diterima via Socket:**
1. Post `/message` API mengembalikan pesan yang *belum di-populate* (sehingga `msg.sender` adalah string ObjectID, bukan objek).
2. `msg.sender._id` pada string menghasilkan `undefined`.
3. Seperti dibahas di atas, `user._id` juga `undefined`.
4. Evaluasi menjadi: `undefined === undefined`. Ini **TRUE**!
5. Karena `isMe` bernilai `true`, semua pesan (*incoming* dan *outgoing* via socket) akan dianggap milik sendiri dan dirender di sebelah **kanan** (`wa-msg.me`).

*(Catatan: Jika halaman di-refresh, API mem-populate `msg.sender` sehingga punya `_id`. Evaluasi: `"65f..." === undefined` menjadi FALSE, dan semua pesan malah pindah ke kiri. Ini sangat mengacaukan layout).*

## 4. Perubahan yang Akan Diimplementasi

Kita hanya perlu mengoreksi pemanggilan ID user di `Chat.tsx` dengan menggunakan *fallback* `(user.id || user._id)` agar konsisten dengan format localStorage.

### [MODIFY] `Frontend/frontend/src/pages/Chat.tsx`

**1. Buat variable pembantu `myId`:**
```typescript
const user = JSON.parse(localStorage.getItem("user") || "{}");
const myId = user.id || user._id; // FIX: Handle JWT payload format
```

**2. Ganti `user._id` di blok Partner Filter:**
```typescript
const partner = conv.participants.find(
  (p: any) => p._id !== myId
);
```

**3. Ganti `user._id` di blok `isMe`:**
```typescript
const isMe =
  msg.sender?._id === myId ||
  (msg.sender as any) === myId;
```

**4. Ganti `user._id` di blok Typing Handler:**
```typescript
socket.emit("typing", { conversationId, userId: myId });
socket.emit("stop_typing", { conversationId, userId: myId });
```

## 5. Tracing Scenarios

### Scenario 1: Menampilkan Header Name

1. `Chat.tsx` mount.
2. `myId` = `"65fxyz"`.
3. Fetch conversation participants: `[{ _id: "65fxyz", name: "User" }, { _id: "777abc", name: "Dr. Crypto" }]`.
4. `partner = participants.find(p => p._id !== "65fxyz")`.
5. Hasilnya objek `"777abc"` (Dr. Crypto).
6. Header menampilkan nama **"Dr. Crypto"** secara akurat.

### Scenario 2: Mengirim dan Menerima Pesan Realtime

1. User ketik pesan lalu tekan Enter.
2. Socket Menerima Pesan Balasan: `msg = { text: "Halo", sender: "777abc" }`.
3. `isMe` evaluation: `msg.sender._id` (undefined) === `"65fxyz"` (FALSE) || `"777abc"` === `"65fxyz"` (FALSE).
4. `isMe` bernilai **FALSE**.
5. Rendering: Pesan masuk div dengan class `wa-msg other`, muncul di sebelah **KIRI**.
6. User ketik pesan lagi "Baik".
7. `isMe` evaluation: `msg.sender` === `"65fxyz"`. Hasilnya **TRUE**.
8. Rendering: Pesan dikirim div dengan class `wa-msg me`, muncul di sebelah **KANAN**.
