# Spec 024: User Profile Popup & Settings

## 1. Prompt & Objective
**Prompt:** "{tolong saya di halaman user nya itu apa bila saya klik itubantu saya keluarkan halaman popup untuk update profile nya nah lalu bisa logout bantu saya }"
**Tujuan (Objective):** Membuat interaksi klik pada avatar/profil user di header Dashboard. Saat diklik, akan muncul popup/modal yang memungkinkan user untuk:
1. Mengubah detail profil (Nama, Email, dan Password baru).
2. Melakukan Logout dari aplikasi.
**Kenapa tujuan ini penting:** Ini adalah pola UX standar (User Profile Dropdown/Modal) di mana pengaturan akun dan tombol keluar dikelompokkan dalam satu menu yang diakses lewat avatar. Selain merapikan header, ini memberikan privasi dan kontrol ke user atas data mereka.

## 2. Analisis Codebase & Architecture
- **Frontend (`Dashboard.tsx` & `Dashboard.css`):**
  Saat ini tombol logout berada terpisah di samping avatar. Kita akan menggabungkannya ke dalam sebuah Modal Popup Settings. Avatar perlu diberikan _onClick handler_.
- **Backend Authentication (`routes/user.js` vs `routes/auth.js`):**
  Di `auth.js` terdapat `PUT /user/:id`, **TETAPI** endpoint ini tidak menggunakan `auth` middleware (rentan/insecure) dan mengizinkan pengubahan `balance` serta `role`. Ini adalah endpoint khusus Admin. 
  **Oleh karena itu**, kita wajib membuat endpoint baru khusus user: `PUT /api/user/profile` di `routes/user.js`. Ini akan berjalan melalui middleware `auth` sehingga user HANYA bisa mengubah datanya sendiri (via `req.user.id`), dan hanya atribut yang diizinkan (Nama, Email, Password).

## 3. Rencana Perubahan (Logic & Code)

### A. Perubahan Backend (`Backend/backend/routes/user.js`)
1. Tambahkan `const bcrypt = require("bcryptjs");`.
2. Buat endpoint baru `router.put("/profile", auth, async (req, res) => { ... })`.
3. Validasi:
   - Ambil `name`, `email`, `password` dari `req.body`.
   - Cek apakah email baru bertabrakan dengan user lain `User.findOne({ email, _id: { $ne: req.user.id } })`.
   - Update `user.name` dan `user.email`.
   - Jika `password` diisi (lenght > 0), lakukan hashing dengan `bcrypt.hash()` lalu timpa `user.password`.
   - Simpan (`user.save()`).
   - Kembalikan response JSON berisi profil terbaru.

### B. Perubahan Frontend (`Frontend/frontend/src/pages/Dashboard.tsx`)
1. Tambahkan state `showProfileModal` (boolean).
2. Tambahkan state form `editName`, `editEmail`, `editPassword`.
3. Pada tag `.avatar` di header, tambahkan `onClick={() => setShowProfileModal(true)}` dan set `cursor: pointer`.
4. Buat blok JSX Modal baru di paling bawah komponen (di luar container absolute agar overlay menutupi semuanya):
   - Form input Name (value terikat ke `editName`).
   - Form input Email (value terikat ke `editEmail`).
   - Form input Password Baru (kosong default).
   - Tombol "Simpan Perubahan" yang men-trigger `handleUpdateProfile`.
   - Pindahkan logika / tombol "Logout" dari header ke dalam modal ini (tombol merah mencolok).
5. Buat fungsi `handleUpdateProfile`:
   - Lakukan `fetch("http://localhost:5000/api/user/profile", { method: "PUT" ... })`.
   - Perbarui tampilan `userName` di header dan isi `localStorage` dengan nama/email baru.

### C. Perubahan Frontend Style (`Frontend/frontend/src/pages/Dashboard.css`)
1. Tambahkan styling untuk modal `.profile-modal-overlay` dan `.profile-modal-box`.
2. Styling form `.profile-form-group` dan styling `.logout-modal-btn`.

## 4. Pseudocode & Tracing
```text
// User clicks Avatar
Action: setShowProfileModal(true)
UI: Modal opens. Pre-fills editName="Budi", editEmail="budi@gmail.com". editPassword="".

// User types new Name "Budi Santoso" and clicks Save
Action: handleUpdateProfile()
Data Sent: { name: "Budi Santoso", email: "budi@gmail.com", password: "" }
Backend:
  Verify token -> userId
  FindUserById(userId)
  user.name = "Budi Santoso"
  user.email = "budi@gmail.com"
  (password skipped because empty)
  user.save()
  Return success
Frontend UI:
  Close Modal
  Update userName state -> Header shows "Selamat Pagi, Budi Santoso"
```

## 5. Manual Testing Plan
1. Login menggunakan akun standar (bukan admin).
2. Di Dashboard, klik avatar (inisial/gradien warna) di kanan atas.
3. Pastikan modal Profil muncul dan meng-cover layar. Form Nama dan Email harus sudah terisi.
4. Ubah Nama Anda, diamkan kolom password, lalu tekan "Simpan".
5. Pastikan toast "Berhasil" muncul, modal tertutup, dan nama di Header langsung berubah.
6. Buka kembali profil, isikan "password baru" dan simpan. 
7. Buka kembali profil, klik tombol Logout merah.
8. Pastikan user sukses diarahkan(ditendang) kembali ke halaman Login.
9. Tes login kembali menggunakan password baru yang di-set pada langkah 6.
