# Spec 027: Admin Dashboard - API Calls Tracking Line Chart

## 1. Prompt & Objective
**Prompt:** "{tolong saya yang dihalaman dashboard admin itu muncul kan juga chart line chart berapa kali memanggil data api nya}"
**Tujuan (Objective):**
Menambahkan sebuah "Line Chart" (Grafik Garis) di halaman Admin Dashboard yang memvisualisasikan tren jumlah pemanggilan (hits) ke API backend setiap harinya.
**Kenapa tujuan ini penting:** Membantu admin menganalisis load server yang sesungguhnya dan traffic penggunaan aplikasi seiring waktu, sesuai standar dashboard "System Management".

## 2. Analisis Codebase & Architecture
- **Kondisi Backend Saat Ini:** Belum ada mekanisme pencatatan traffic/hit rate ke sistem Express.js (`server.js`).
- **Kondisi Frontend Saat Ini (`DashboardAdmin.tsx`):** Pada task sebelumnya (Spec 026), Line Chart dicopot karena data dummy-nya tidak digunakan. Kita perlu mengembalikannya dengan library `react-chartjs-2`, tapi kali ini dirender dari data real.
- **Kebutuhan Sistem:**
  1. **MongoDB Model Baru:** `ApiStat`. Model ini akan menyimpan hitungan pemanggilan API per tanggal (sehingga tidak membebani database dengan membuat jutaan row per request, melainkan menggunakan pola inkremental harian / *daily bucket pattern*).
  2. **Express Middleware:** Sebuah custom middleware global di `server.js` yang mengeksekusi `$inc` ke dokumen `ApiStat` pada tanggal hari ini setiap kali route `/api/` diakses.
  3. **Admin Route Update:** Tambahkan agregasi/pengambilan koleksi `ApiStat` 7 hari terakhir ke dalam respon `GET /api/admin/dashboard-stats`.

## 3. Rencana Perubahan (Logic & Code)

### A. Perubahan Backend (`Backend/backend/`)
1. **Model Tracker Baru (`models/ApiStat.js`):**
   ```javascript
   const mongoose = require("mongoose");
   const apiStatSchema = new mongoose.Schema({
     date: { type: String, required: true, unique: true }, // Format: YYYY-MM-DD
     count: { type: Number, default: 0 }
   });
   module.exports = mongoose.model("ApiStat", apiStatSchema);
   ```
2. **Global Middleware (`server.js`):**
   Suntikkan pada pipeline middleware teratas di `server.js` sebelum router utama:
   ```javascript
   app.use("/api", async (req, res, next) => {
     // Run tracking asynchronously (non-blocking)
     res.on("finish", async () => {
       try {
         const ApiStat = require("./models/ApiStat");
         const today = new Date().toISOString().split("T")[0]; // "2026-03-20"
         await ApiStat.findOneAndUpdate(
           { date: today },
           { $inc: { count: 1 } },
           { upsert: true, new: true }
         );
       } catch (err) { console.error("API Stat tracking failed"); }
     });
     next();
   });
   ```
3. **Endpoint Analytics (`routes/admin.js`):**
   Pada route `GET /dashboard-stats`, tambahkan fetch data log:
   ```javascript
   const apiLogs = await ApiStat.find().sort({ date: -1 }).limit(7);
   // Balikkan urutan agar tanggal urut dari kiri ke kanan di chart
   const apiChartData = apiLogs.reverse(); 
   ```
   Tambahkan `apiChartData` ke balikan response ke DashboardAdmin.

### B. Perubahan Frontend (`Frontend/frontend/src/pages/admin/DashboardAdmin.tsx`)
1. Import kembali `Line`, `LineElement`, dan `PointElement` di `chart.js` & `react-chartjs-2`.
2. Tambahkan React state `apiChartData`, dan tangkap nilainya di dalam block pemanggilan Promise ( `fetchStats` ).
3. Siapkan fungsi formatter untuk Line Chart `formatLineChartData()` menggunakan `apiChartData.date` sebagai label X dan `apiChartData.count` sebagai garis Y.
4. Render Line Chart ke DOM di tepat **bawah** `.charts-grid` (menempati width yang membentang penuh `100%`) dengan border container `chart-box`.

### C. Peruntukan CSS (`Frontend/frontend/src/pages/admin/DashboardAdmin.css`)
- Tambahkan class pembungkus spesifik `.line-chart-container` yang memiliki properti:
  `width: 100%; margin-top: 24px; height: 300px;` sehingga grafiknya lebar dan rapi seperti Dashboard analytics modern.
