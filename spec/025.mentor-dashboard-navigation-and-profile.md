# Spec 025: Mentor Dashboard Navigation & Profile Updates

## 1. Prompt & Objective
**Prompt:** "{bantu saya dihalaman menotor yang di bagian halaman dashboard nya itu bantu saya agar apa bila saya klik yang bagian total siswa nya maka dia akan ke halaman pantau siswa apa bila saya klik inbox chat maka akanke halaman inbox dan juga bantu saya apa bila saya klik di profile maka dia akan keluar data nya lalu saya bisa melakukan edit bantu saya saya bisa edit semua pengalaman mentor dll bantu saya }"
**Tujuan (Objective):**
1. Membuat kartu statistik "Total Siswa" di Mentor Dashboard dapat diklik untuk menavigasi ke tab "Pantau Siswa".
2. Membuat kartu statistik "Inbox Chat" di Mentor Dashboard dapat diklik untuk menavigasi ke tab "Inbox Q&A".
3. Membuat Avatar Mentor di pojok kanan atas dapat diklik untuk memunculkan modal profil yang komprehensif, memungkinkan update pada field Nama, Email, Password, serta Atribut Khusus Mentor (Pengalaman, Spesialisasi, Bio, dan Gaya Mengajar).

**Kenapa tujuan ini penting:** Mempercepat workflow mentor lewat shortcut navigasi dan memberikan kontrol penuh ke mentor untuk memelihara profil publiknya langsung dari dashboard.

## 2. Analisis Codebase & Architecture
- **Stat Cards (`frontend/src/pages/mentor/MentorDashboard.tsx`):**
  Saat ini hanya dirender statis. Kita cukup menyematkan event `onClick={() => setMenu("siswa")}` dan `onClick={() => setMenu("qna")}` pada card bersangkutan, serta memberikan efek hover (pointer cursor) di file CSS-nya.
- **Backend User Profile Endpoint (`backend/routes/user.js`):**
  Pada task sebelumnya (Spec 024) kita menambahkan endpoint `PUT /api/user/profile`. Saat ini endpoint tersebut hanya menerima `name`, `email`, dan `password`.
  Kita perlu me-whitelist field mentor: `specialization`, `bio`, `experience`, dan `style` agar mentor dapat menyimpannya.
- **Mentor Profile Modal (`MentorDashboard.tsx`):**
  Akan dibuat serupa dengan milik User Dashboard (Spec 024) tetapi dengan input form yang jauh lebih lengkap khusus parameter mentor di atas.

## 3. Rencana Perubahan (Logic & Code)

### A. Perubahan Backend (`Backend/backend/routes/user.js`)
1. Buka route `PUT /profile`.
2. Extract argumen baru: `const { name, email, password, specialization, bio, experience, style } = req.body;`.
3. Tambahkan validasi dan update model:
   ```javascript
   if (specialization !== undefined) user.specialization = specialization;
   if (bio !== undefined) user.bio = bio;
   if (experience !== undefined) user.experience = experience;
   if (style !== undefined) user.style = style;
   ```
4. Update `res.json` untuk juga mengembalikan field tersebut di objek `data`.
5. Di handler `GET /me`, pastikan field-field ini juga ikut di-return meskipun belum spesifik ter-define. Bisa tambahkan mapping:
   ```javascript
   specialization: user.specialization,
   bio: user.bio,
   experience: user.experience,
   style: user.style
   ```

### B. Perubahan Frontend (`Frontend/frontend/src/pages/mentor/MentorDashboard.tsx`)
1. Pada `useEffect` inisialisasi awal `GET /me`, tarik data `bio`, `specialization`, `experience`, dan `style` lalu simpan dalam React state (buat statenya terlebih dahulu, misal `const [editBio, setEditBio] = useState("")`, dll).
2. Modifikasi JSX Stat Card:
   - `<div className="stat-card" onClick={() => setMenu("siswa")} style={{ cursor: "pointer" }}>`
   - `<div className="stat-card" onClick={() => setMenu("qna")} style={{ cursor: "pointer" }}>`
3. Tambahkan `showProfileModal` state dan event `onClick` pada class `.avatar`.
4. Rangkai form Profile Backend Modalnya:
   - Nama, Email, Password (seperti biasa).
   - `specialization` (contoh: "Technical Analysis, Day Trading").
   - `experience` (input number, dalam satuan Tahun).
   - `bio` (textarea).
   - `style` (contoh: "Scalping, Trend Following").
5. Saat submit `handleUpdateProfile`, sertakan semua argumen di payload JSON. Update UI apabila sukses (200 OK).
6. Pindahkan tombol Logout ke dalam footer modal agar konsisten dengan `Dashboard.tsx`.

### C. Perubahan Frontend Style (`Frontend/frontend/src/pages/mentor/MentorDashboard.css`)
1. Salin blok arsitektur modal profile overlay yang telah dibuat di User Dashboard. Sesuaikan styling-nya (seperti padding dan scrollbar jika form terlalu panjang).
2. Berikan cursor pointer pada `.stat-card:hover`.
