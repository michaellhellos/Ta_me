## 5. Logic / Pseudocode

### 5.1 Backend Redeem (nilaiController.js)

```
FUNCTION redeemXp(userId, amountXp):
  IF amountXp == 40: reward = 5000
  ELSE IF amountXp == 80: reward = 12000
  ELSE: return ERROR "Opsi redeem tidak valid"

  totalEarned = SUM(Nilai.find({userId}).score)
  user = User.findById(userId)
  availableXp = totalEarned - user.redeemedXp

  IF availableXp < amountXp: return ERROR "XP tidak mencukupi"

  user.redeemedXp += amountXp
  user.balance += reward
  user.save()

  return SUCCESS { redeemedXp, balance }
```

### 5.2 Frontend Redeem Button Logic

```
Button "40 XP → $5,000":
  disabled = availableXp < 40 OR isRedeeming
  onClick → handleRedeem(40)

Button "80 XP → $12,000":
  disabled = availableXp < 80 OR isRedeeming
  onClick → handleRedeem(80)
```

## 6. Tracing Scenarios

### Scenario 1: User mengerjakan quiz dan kembali ke list (verifikasi live update)

1. State awal: `nilaiMap = { "m1": { score: 8 } }`, `totalEarnedXp = 8`, `completedMateri = [m1]`, `pendingMateri = [m2, m3]`
2. User klik quiz "m2" → `setSelectedMateri(m2)` → quiz view opens
3. User jawab semua → klik Submit → `submitQuiz()` → score 6 XP
4. `setNilaiMap(prev => ({...prev, "m2": { score: 6 }}))` → `nilaiMap = { "m1": {score:8}, "m2": {score:6} }`
5. User klik "Kembali ke Daftar Quiz" → `setSelectedMateri(null)` → re-render
6. React recalculates:
   - `totalEarnedXp = 8 + 6 = 14`
   - `completedMateri = [m1, m2]` (length: 2)
   - `pendingMateri = [m3]` (length: 1)
7. XP point card menampilkan: **⭐ 14 XP | 2 selesai | 1 tersisa** ← langsung terupdate ✅

### Scenario 2: User redeem 40 XP → $5,000

1. State: `totalEarnedXp = 50`, `redeemedXp = 0`, `availableXp = 50`
2. User klik XP Summary → modal opens
3. Button "40 XP → $5,000" **enabled** (50 >= 40), Button "80 XP → $12,000" **disabled** (50 < 80)
4. User klik "40 XP → $5,000"
5. `POST /api/nilai/redeem` → body: `{ userId: "abc", amountXp: 40 }`
6. Backend: `amountXp === 40` → `reward = 5000`
7. Backend: check `50 - 0 >= 40` → ✅
8. Backend: `user.redeemedXp = 40`, `user.balance += 5000` → save
9. Response: `{ success: true, message: "Berhasil redeem 40 XP menjadi $5000", redeemedXp: 40 }`
10. Frontend: `setRedeemedXp(40)` → `availableXp = 50 - 40 = 10`
11. XP Summary menampilkan "10 XP Tersedia"
12. Toast hijau: "Berhasil redeem 40 XP menjadi $5000"

### Scenario 3: User redeem dengan XP tidak cukup

1. `availableXp = 30`
2. Modal opens → Button "40 XP" **disabled** (30 < 40), Button "80 XP" **disabled** (30 < 80)
3. User tidak bisa klik tombol manapun

### Scenario 4: User redeem 80 XP → $12,000

1. `totalEarnedXp = 90`, `redeemedXp = 0`, `availableXp = 90`
2. Button "80 XP → $12,000" **enabled**
3. User klik → `POST /api/nilai/redeem` → body: `{ amountXp: 80 }`
4. Backend: `amountXp === 80` → `reward = 12000`
5. Backend: `user.redeemedXp = 80`, `user.balance += 12000`
6. Frontend: `availableXp = 90 - 80 = 10`
