## 3. Kenapa Tujuan Ini?

### 3.1 BUG KRITIS: Portfolio TIDAK Muncul di Dashboard

**`GET /api/user/me`** (backend `routes/user.js` line 16-21):

```javascript
res.json({
  id: user._id,
  name: user.name,
  email: user.email,
  balance: user.balance,
  // ❌ TIDAK ADA portfolio!
});
```

Dashboard.tsx (line 53-54):
```tsx
setPortfolio(Array.isArray(userData?.portfolio) ? userData.portfolio : []);
// userData.portfolio === undefined → portfolio = [] → "Kamu belum membeli kripto 🚀"
```

Artinya **walaupun user sudah beli kripto, portofolio SELALU kosong** di dashboard karena backend tidak mengirim data portfolio.

### 3.2 Dua Sistem Portfolio yang Konflik

Ada DUA mekanisme portfolio yang berbeda:

| # | File | Storage | Dipakai untuk |
|---|------|---------|---------------|
| 1 | `routes/transaction.js` | Embedded di User doc (`user.portfolio[]`) | **LAMA** — `userId: req.userId` (typo, harusnya `req.user.id`) |
| 2 | `routes/trade.js` | Separate `Portfolio` collection | **BARU** — dipanggil oleh `Simulasi.tsx` |

**Sistem yang aktif** adalah `trade.js` (dipanggil oleh `POST /api/trade/buy` dan `POST /api/trade/sell`). `transaction.js` route di `server.js` registered sebagai `/api/transaction` tapi `Simulasi.tsx` pakai `/api/trade/*`.

`GET /api/user/me` tidak return data dari **KEDUA** sistem. Solusi: fetch `Portfolio` collection di endpoint `/me` response.

### 3.3 BUG: Buy Price Overwrite

`trade.js` line 176-178 — saat beli LEBIH dari coin yang sudah dimiliki:

```javascript
if (portfolio) {
  portfolio.quantity += qty;
  portfolio.price = coin.current_price;  // ❌ OVERWRITE, bukan weighted average!
  await portfolio.save();
}
```

**Contoh masalah:**
1. Beli 1 BTC @ $50,000 → portfolio: `{ price: 50000, quantity: 1 }`
2. Beli 1 BTC @ $60,000 → portfolio: `{ price: 60000, quantity: 2 }` ❌
3. Seharusnya weighted avg: `(50000*1 + 60000*1) / 2 = $55,000`
4. Dashboard menghitung profit berdasarkan `price` ini → **profit salah**

### 3.4 Tidak Ada Dollar Profit & Total P/L

Dashboard saat ini menampilkan:
- ✅ `totalValue` = `currentPrice * quantity` (benar)
- ✅ `profitPercent` = `(current - buy) / buy * 100` (benar JIKA price benar)
- ❌ `profitDollar` = TIDAK ADA (user tidak tahu berapa $ untung/rugi)
- ❌ `totalProfit` = TIDAK ADA (berapa total keuntungan seluruh portfolio?)
- ❌ `totalInvested` = TIDAK ADA (berapa total modal yang dikeluarkan?)

## 10. Catatan Implementasi

### Tidak Perlu Dependencies Baru
- Semua library yang dibutuhkan sudah installed
- Hanya tambah `require("../models/Portfolio")` di `user.js`

### Backward Compatibility
- `transaction.js` route (lama) tidak diubah/dihapus
- Portfolio model tidak berubah schema — hanya cara `price` diset yang berubah
- Dashboard.tsx tetap handle `portfolio = []` (empty state)

### Precision
- Dollar values: 2 decimal places (`toFixed(2)`)
- Percent values: 2 decimal places (`toFixed(2)`)
- Quantity: 6 decimal places (crypto standard)
