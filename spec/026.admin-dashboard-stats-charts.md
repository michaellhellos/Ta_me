# Spec 026: Admin Dashboard UI Redesign & Top Students Charts

## 1. Prompt & Objective
**Prompt:** "{bantu saya untuk agar tampilan yang di bagian halaman admin nya itu tolong saya yang di bagian dashbaoard nya itu tolong perbaiki na habis itu tolong saya kasih juga chart chart yang di halaman dashboard nya siapa siswa yang paling sering beli siapa siswa yang sering jual bantu saya dan juga siapa siswa yang aktif untuk menjawab soal soal bantu saya }"
**Tujuan (Objective):** 
1. Memperbaiki dan mempercantik antarmuka (UI) khusus bagian Dashboard Admin (`DashboardAdmin.tsx` dan `DashboardAdmin.css`) menggunakan prinsip "Intentional Minimalism" (contoh: glassmorphism, warna yang lebih modern, whitespace yang presisi).
2. Menghapus chart hardcode ("Aktivitas Transaksi Mingguan") dan menggantinya dengan 3 Chart analitik nyata:
   - **Top Buyers:** Siswa yang paling sering melakukan transaksi "BUY" koin kripto.
   - **Top Sellers:** Siswa yang paling sering melakukan transaksi "SELL" koin kripto.
   - **Top Quiz Active:** Siswa yang paling aktif mengerjakan/menjawab soal materi (berdasarkan riwayat skor `Nilai`).

Keuntungan: Membantu sisi admin/bisnis mengidentifikasi "Power Users" dalam ekosistem Kripto-Z, baik dalam literasi edukasi maupun simulasi trading.

## 2. Analisis Codebase & Architecture
- **Struktur UI Saat Ini (`Frontend/frontend/src/pages/admin/DashboardAdmin.tsx`):**
  Menggunakan template grid dengan 4 card stat dan 1 chart "Line" dummmy dari Chart.js.
- **Model Backend (`backend/models/Transaction.js` & `backend/models/Nilai.js`):**
  - Transaksi kripto disimpan di koleksi `Transaction`. Terdapat validasi `type: "BUY" | "SELL"` dan direferensikan ke `userId`.
  - Progres quiz / evaluasi akhir disimpan di koleksi `Nilai`, yang memiliki `userId`, `score`, dan `totalSoal`.
- **Backend Routing:** 
  Belum ada endpoint khusus admin yang melakukan operasi agregasi (Aggregation Pipeline) untuk menghasilkan metrik Top 5 ini. Saya perlu merancang route baru `/api/admin/dashboard-stats` atau menambahkannya ke `server.js` untuk kebersihan arsitektur.

## 3. Rencana Perubahan (Logic & Code)

### A. Perubahan Backend (`backend/server.js` & `backend/routes/admin.js`)
1. Buat file baru: `backend/routes/admin.js`.
2. Pasang router endpoint `GET /dashboard-stats`.
3. Lakukan 3 Agregasi MongoDB (`Promise.all`):
   - **Top Buyers Pipeline:**
     ```javascript
     Transaction.aggregate([
       { $match: { type: "BUY" } },
       { $group: { _id: "$userId", count: { $sum: 1 }, volume: { $sum: "$total" } } },
       { $sort: { count: -1 } },
       { $limit: 5 },
       { $lookup: { from: "users", localField: "_id", foreignField: "_id", as: "user" } },
       { $unwind: "$user" },
       { $project: { name: "$user.name", email: "$user.email", count: 1, volume: 1 } }
     ])
     ```
   - **Top Sellers Pipeline:** (mirip seperti di atas, ubah `$match: { type: "SELL" }`).
   - **Top Quiz Takers Pipeline:**
     ```javascript
     Nilai.aggregate([
       { $group: { _id: "$userId", quizzesTaken: { $sum: 1 }, maxScore: { $max: "$score" } } },
       { $sort: { quizzesTaken: -1 } },
       { $limit: 5 },
       { $lookup: { from: "users", localField: "_id", foreignField: "_id", as: "user" } },
       { $unwind: "$user" },
       { $project: { name: "$user.name", quizzesTaken: 1, maxScore: 1 } }
     ])
     ```
4. Di file `server.js`, daftarkan route baru tersebut: `app.use("/api/admin", require("./routes/admin"));`.

### B. Perubahan Frontend (`Frontend/frontend/src/pages/admin/DashboardAdmin.tsx`)
1. Hapus state `chartData` dummy.
2. Tambahkan HTTP Fetch di dalam `useEffect` memanggil `/api/admin/dashboard-stats`.
3. Simpan balikan agregasi backend ke State (misalnya `topBuyers`, `topSellers`, dan `topQuiz`).
4. Import komponen grafis dari react-chartjs-2 (misal: `Bar` chart).
5. Definisikan 3 kanvas Bar Chart berjajar menggunakan CSS Grid/Flexbox di posisi chart box yang digantikan tadi:
   - Chart 1: "Top 5 Siswa Sering Beli"
   - Chart 2: "Top 5 Siswa Sering Jual"
   - Chart 3: "Siswa Paling Rajin Quiz"

### C. Perubahan Frontend Style (`Frontend/frontend/src/pages/admin/DashboardAdmin.css`)
1. Re-engineer gaya admin CSS untuk menghapus gaya primitif bawaan.
2. Aplikasikan palet warna app-level `--card-bg`, `--card-blur`, neon glows, dan border-radius modern yang seragam dengan user dashboard.
3. Buat layout `.charts-grid` untuk menampung 3 chart (satu baris, atau dibungkus rapi sesuai ukuran layar).
