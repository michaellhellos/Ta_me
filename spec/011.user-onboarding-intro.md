---

## 2. Tujuan yang Dipahami

Membuat **halaman intro/onboarding walkthrough** yang:
1. Muncul **hanya pada login pertama** (first-time user)
2. Berupa **multi-step overlay** yang menjelaskan setiap fitur utama di dashboard
3. Ada tombol **"Lanjut"** (next), **"Lewati"** (skip), dan **"Selesai"** (finish)
4. Setelah selesai atau di-skip, **tidak muncul lagi** di login berikutnya
5. UI bersifat overlay di atas dashboard (bukan halaman terpisah)

---

## 3. Kenapa Fitur Ini Dibutuhkan

Saat ini:
- User baru langsung masuk ke dashboard tanpa panduan
- Dashboard punya 5 tab (Beranda, Simulasi, Belajar, Komunitas, Mentor) + fitur balance, portfolio, market — bisa membingungkan user baru
- Tidak ada penjelasan apa itu "Virtual Equity", cara trading, atau fitur yang tersedia
- Tidak ada mekanisme tracking apakah user sudah pernah login sebelumnya

## 4. Codebase Analysis

### 4.1 Auth Flow

```
Login.tsx
  → POST /api/auth/login
  → Response: { token, user: { _id, name, email, role, balance } }
  → localStorage.setItem("token", data.token)
  → localStorage.setItem("user", JSON.stringify(data.user))
  → localStorage.setItem("userId", data.user._id)
  → navigate("/dashboard")    ← USER LANDS HERE
```

### 4.2 Dashboard Structure (5 tabs)

```
Dashboard.tsx
├── <header>              → Greeting, Avatar, Logout
├── <main>
│   ├── "beranda"         → Total Asset, P/L, Portfolio, Market
│   ├── "simulasi"        → <Simulasi /> — Buy/Sell crypto
│   ├── "belajar"         → <Belajar /> — Educational content
│   ├── "komunitas"       → <Komunitas /> — Forum
│   └── "mentor"          → <Ai /> — Mentor list + schedules
└── <nav className="bottom-nav">  → Tab icons at bottom
```

### 4.3 Key Files

| File | Purpose |
|------|---------|
| `App.tsx` | Routes: `/` (Login), `/dashboard` (Dashboard), etc. |
| `Login.tsx` | Auth form → stores token/user → navigates by role |
| `Dashboard.tsx` | Main user dashboard with 5 tab menu |
| `Dashboard.css` | Existing dashboard styles |
| User model (`user.js`) | No `hasSeenIntro` field |

### 4.4 Tracking Approach

**localStorage-based** (tidak perlu backend):
- Key: `kripto_z_intro_done`
- Set saat user click "Selesai" atau "Lewati"
- Check on Dashboard mount: jika belum ada → tampilkan intro

**Kenapa bukan backend?**
- Simpler, no model/route changes needed
- If user clears localStorage, they just see the intro again (harmless)
- Consistent with how this project already tracks user state (token, user, userId all in localStorage)

## 6. Logic / Pseudocode

### 6.1 Onboarding Component

```typescript
interface OnboardingProps {
  onComplete: () => void;
}

const steps = [
  {
    icon: "🎉",
    title: "Selamat Datang di Kripto-Z!",
    desc: "Simulasi trading crypto tanpa risiko. Mari kenali fitur-fiturnya.",
  },
  {
    icon: "💰",
    title: "Saldo Virtual Rp 1.000.000",
    desc: "Kamu mendapat saldo virtual untuk membeli crypto dan belajar trading.",
  },
  {
    icon: "📈",
    title: "Simulasi Trading",
    desc: "Beli dan jual crypto dengan harga real-time. Latih strategi tradingmu!",
  },
  {
    icon: "📚",
    title: "Belajar & Komunitas",
    desc: "Akses materi edukasi dan diskusi dengan trader lain di forum komunitas.",
  },
  {
    icon: "🎓",
    title: "Mentor Profesional",
    desc: "Konsultasi dengan mentor berpengalaman dan ikuti jadwal mentoring.",
  },
];

const Onboarding = ({ onComplete }: OnboardingProps) => {
  const [step, setStep] = useState(0);

  const isLast = step === steps.length - 1;

  const handleNext = () => {
    if (isLast) {
      onComplete();
    } else {
      setStep(step + 1);
    }
  };

  return (
    <div className="onboarding-overlay">
      <div className="onboarding-card">
        <div className="onboarding-icon">{steps[step].icon}</div>
        <h2>{steps[step].title}</h2>
        <p>{steps[step].desc}</p>

        {/* Progress Dots */}
        <div className="onboarding-progress">
          {steps.map((_, i) => (
            <div key={i} className={`dot ${i === step ? "active" : ""}`} />
          ))}
        </div>

        {/* Actions */}
        <div className="onboarding-actions">
          <button className="skip-btn" onClick={onComplete}>
            Lewati
          </button>
          <button className="next-btn" onClick={handleNext}>
            {isLast ? "Mulai Sekarang! 🚀" : "Lanjut →"}
          </button>
        </div>
      </div>
    </div>
  );
};
```

## 7. Tracing Scenarios

### Scenario 1: First-time user login

1. User baru register → login → navigate to `/dashboard`
2. Dashboard mounts → checks `localStorage.getItem("kripto_z_intro_done")` → **null**
3. `setShowIntro(true)` → overlay muncul
4. Step 0: "Selamat Datang!" → user klik "Lanjut"
5. Step 1: "Saldo Virtual" → user klik "Lanjut"
6. Step 2: "Simulasi Trading" → user klik "Lanjut"
7. Step 3: "Belajar & Komunitas" → user klik "Lanjut"
8. Step 4: "Mentor" → user klik "Mulai Sekarang! 🚀"
9. `onComplete()` → `localStorage.setItem("kripto_z_intro_done", "true")` → `setShowIntro(false)`
10. Overlay hilang, dashboard normal

### Scenario 2: Returning user

1. User login lagi → navigate to `/dashboard`
2. Dashboard mounts → checks `localStorage.getItem("kripto_z_intro_done")` → **"true"**
3. `showIntro` stays false → **no overlay**
4. Dashboard normal

### Scenario 3: User clicks "Lewati" at step 2

1. Step 0 → "Lanjut" → Step 1 → "Lanjut" → Step 2
2. User klik "Lewati"
3. `onComplete()` → `localStorage.setItem("kripto_z_intro_done", "true")`
4. Overlay hilang, dashboard normal
5. Next login → **no overlay**

### Scenario 4: User clears browser data

1. User clear localStorage
2. Login lagi → Dashboard mounts → `kripto_z_intro_done` = null
3. Intro muncul lagi (harmless, bisa di-skip)
