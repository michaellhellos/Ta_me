## 1. Prompt / Task

```
bantu saya di halaman user nya yang di begian belajar itu kan dia ada quiz bantu saya apa bila user berhasil mendapat kan jadwan benar misal saya menjawab pertanyaan nya dan selesai muncul nilai nya itu kan berdasarkan xp nah itu kan di bawah nya muncul nilai nah apa bila saya tekan sumbit maka dia akan kembali ke halaman yang quiz itu lalu bantu saya tambhakan di atas nya yang qiz itu meujukan nilai dari beberapa soal yang sudah di kerjakan dan soal yang sudah di kerjakan dan ada nilai nya itu maka soal itu akan mengilang bantu saya saji tidak ada opsi coba lagi
```

---

## 2. Tujuan yang Dipahami

Berdasarkan prompt, terdapat dua kebutuhan utama di halaman **Belajar** (bagian Quiz):
1. **Memisahkan Quiz yang Sudah Dikerjakan:** Di atas daftar quiz yang tersedia, tambahkan bagian ("Riwayat Quiz / Quiz Selesai") yang hanya menampilkan daftar quiz yang sudah dikerjakan beserta nilainya (XP).
2. **Menghilangkan Opsi Coba Lagi (No Retake):** Quiz yang sudah dikerjakan tidak boleh ada di daftar "Quiz Tersedia", sehingga tidak bisa di-klik atau dikerjakan ulang. Selain itu, tombol "Coba Lagi" yang muncul tepat sesudah user submit jawaban juga harus dihapus.

---

## 3. Kenapa Perubahan Ini Dibutuhkan

| Masalah Saat Ini | Impact / Alasan Perubahan |
|---|---|
| Semua quiz (baik yang sudah dikerjakan maupun belum) tampil di list yang sama dan bisa di-klik. | User bisa dengan mudah mengerjakan ulang quiz yang sama berulang kali (spam XP), sehingga merusak keadilan sistem poin/XP. |
| Tersedia tombol "Coba Lagi" saat user selesai mengerjakan quiz. | Memudahkan user untuk sekadar menebak-nebak jawaban secara brute-force tanpa batas waktu. |
| Tidak ada bagian riwayat yang rapi di halaman utama. | Memisahkan "Quiz Selesai" dan "Quiz Tersedia" akan membuat antarmuka lebih bersih dan progres belajar lebih terlihat jelas. |

## 4. Codebase Analysis

### 4.1. Current UI & State (Belajar.tsx)
- Ada array `materiList` yang berisi semua materi/quiz (`GET /api/materi/materi`).
- Ada object `nilaiMap` yang memetakan `materiId` ke record nilai (`{ score, totalSoal }`) hasil dari `GET /api/nilai/user/:userId`.
- Loop rendering saat ini menggabungkan semua materi: `materiList.map((materi) => ...)` (baris ~300).
- Jika pada materi tersebut ada record di `nilaiMap`, maka hanya ditambahkan tulisan badge `✅ <score> XP`. Tapi materi tersebut tetap bisa diklik untuk mengerjakan ulang.
- Tampilan setelah quiz disubmit merender blok `<div className="quiz-score">` (baris ~414). Terdapat tombol `<button className="quiz-retry-btn">Coba Lagi</button>`.

### 4.2. Logika Perubahan (Requirements)
Perlu dilakukan filtering pada daftar materi:
1. `completedMateri = materiList.filter(m => nilaiMap[m._id])` 
2. `pendingMateri = materiList.filter(m => !nilaiMap[m._id])`

UI di daftar tab "quiz" perlu dipecah:
- Menampilkan section **"Quiz Selesai"** (menggunakan `completedMateri`). Menampilkan judul clan XP, tetapi card ini **tidak** memiliki trigger `onClick` (`setSelectedMateri`).
- Menampilkan section **"Quiz Tersedia"** (menggunakan `pendingMateri`). Menggunakan komponen `belajar-card` yang lama agar bisa di-klik.

**Perubahan Rendering (sebelumnya loop semua materi, jadikan 2 bagian):**
```tsx
{/* =============== QUIZ TAB =============== */}
{activeTab === "quiz" && !selectedMateri && (
  <>
    {/* Total XP summary ... */}
    
    <div className="belajar-content-wrapper">
      {/* 1. Riwayat Quiz Selesai */}
      {!loading && completedMateri.length > 0 && (
        <div className="completed-quizzes-section">
          <h3 className="section-title">✅ Quiz Selesai</h3>

                <span className="cq-score">+{nilaiMap[materi._id].score} XP</span>
              </div>
            ))}
          </div>

      )}

      {/* 2. Quiz Tersedia */}
      {!loading && (
        <div className="pending-quizzes-section">
          <h3 className="section-title">📝 Quiz Tersedia</h3>

              <p>Nantikan update materi edukasi dari mentor!</p>

            <div className="belajar-list">
              {pendingMateri.map((materi) => (
                <div
                  key={materi._id}
                  className="belajar-card"
                  onClick={() => {
                    setSelectedMateri(materi);
                    setAnswers([]);
                    setXp(null);
                    setSubmitted(false);
                  }}
                >
                  <h4>{materi.title}</h4>
                  <p>{materi.summary}</p>
                  <div className="belajar-card-footer">
                    <span className="quiz-badge">
                      <FileQuestion />
                      {materi.quizzes.length} Soal
                    </span>
                  </div>
                </div>
              ))}
            </div>
          )}
        </div>
      )}
    </div>
  </>
)}
```

## 6. Tracing Scenarios

### Scenario 1: State Awal (User sudah pernah mengisi 1 Quiz)
1. Fetches data: `materiList` berisi 3 materi ([M1, M2, M3]), `nilaiMap` berisi data untuk M1.
2. Filter: `completedMateri` = [M1], `pendingMateri` = [M2, M3].
3. UI Render:
   - Bagian **"✅ Quiz Selesai"** muncul dengan 1 baris item "M1 ... +10 XP". Tidak ada *hover effect* pointer / tidak ada fungsi click materi.
   - Bagian **"📝 Quiz Tersedia"** merender card utuh M2 dan M3.

### Scenario 2: Menyelesaikan Quiz Baru
1. User meng-klik materi M2 dari daftar "Quiz Tersedia".
2. Quiz view terbuka. User submit quiz.
3. Fungsi `submitQuiz` dipanggil, fetch endpoint POST `/api/nilai`. Score misalnya 8 XP.
4. `setNilaiMap` langsung meng-update `nilaiMap` agar data M2 tercatat (`nilaiMap[M2._id] = {score: 8}`).
5. View berganti ke success screen (score Xp).
6. **OBSERVASI:** Tombol `Coba Lagi` sudah lenyap, yang ada adalah `Kembali ke Daftar Quiz`.
7. User klik `Kembali ke Daftar Quiz`. State reset (`selectedMateri` = null).
8. Render kembali daftar. Sekarang `nilaiMap` menyimpan M1 dan M2.
9. Filter calculation aktif: `completedMateri` = [M1, M2], `pendingMateri` = [M3].
10. UI M2 otomatis lenyap dari daftar bawah (Quiz Tersedia) dan muncul di daftar atas (Quiz Selesai) dengan niai +8 XP.

7. Verifikasi bahwa quiz yang barusan dikerjakan SUDAH HILANG dari bagian "Quiz Tersedia".
