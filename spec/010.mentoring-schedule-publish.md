## 2. Tujuan yang Dipahami

Membuat fitur **Jadwal Mentoring** dimana:
1. **Mentor** dapat **membuat dan publish** jadwal mentoring (tanggal, waktu, judul/topik, link Zoom)
2. **User (siswa)** dapat **melihat** jadwal mentoring yang sudah di-publish pada tab "Mentor" mereka
3. Jadwal yang sudah lewat tidak ditampilkan (atau ditandai berbeda)
4. Mentor dapat mengelola (hapus) jadwal yang sudah dibuat

---

## 6. Logic / Pseudocode

### 6.1 Backend: `POST /api/schedule`

```javascript
router.post("/", auth, async (req, res) => {
  // 1. Verify user is mentor
  if (req.user.role !== "mentor") return res.status(403)

  // 2. Validate required fields
  const { title, date, zoomLink } = req.body
  if (!title || !date || !zoomLink) return res.status(400)

  // 3. Create schedule
  const schedule = new Schedule({
    mentorId: req.user._id,
    title,
    description: req.body.description || "",
    date: new Date(date),
    zoomLink
  })

  await schedule.save()
  res.json({ success: true, data: schedule })
})
```

### 6.2 Backend: `GET /api/schedule/upcoming`

```javascript
router.get("/upcoming", async (req, res) => {
  // Return published schedules where date >= now
  const schedules = await Schedule.find({
    isPublished: true,
    date: { $gte: new Date() }
  })
    .populate("mentorId", "name specialization")
    .sort({ date: 1 })

  res.json({ success: true, data: schedules })
})
```

### 6.3 Frontend: MentorDashboard Schedule Form

```typescript
// State
const [schedules, setSchedules] = useState([]);
const [showForm, setShowForm] = useState(false);
const [form, setForm] = useState({
  title: "",
  description: "",
  date: "",
  time: "",
  zoomLink: ""
});

// Fetch on mount
useEffect(() => {
  fetchSchedules();
}, []);

const fetchSchedules = async () => {
  const token = localStorage.getItem("token");
  const res = await axios.get("/api/schedule/mentor", {
    headers: { Authorization: `Bearer ${token}` }
  });
  setSchedules(res.data.data || []);
};

// Submit
const handleSubmit = async () => {
  const dateTime = new Date(`${form.date}T${form.time}`);
  await axios.post("/api/schedule", {
    title: form.title,
    description: form.description,
    date: dateTime.toISOString(),
    zoomLink: form.zoomLink
  }, { headers: { Authorization: `Bearer ${token}` }});

  fetchSchedules();
  setShowForm(false);
  setForm({ title: "", description: "", date: "", time: "", zoomLink: "" });
};
```

## 7. Tracing Scenarios

### Scenario 1: Mentor publish jadwal baru

1. Mentor login → MentorDashboard → "Sesi Live" tab
2. Click "+ Buat Jadwal Baru" → form muncul
3. Isi: title="Analisis BTC", date="2026-03-01", time="19:00", zoomLink="https://zoom.us/j/123"
4. Click "Publish" → POST `/api/schedule`
5. Backend: validate mentor role ✅ → create Schedule doc → return success
6. Frontend: fetchSchedules() → jadwal muncul di list
7. Stats card "Live Now" atau schedule count updated

### Scenario 2: User melihat jadwal

1. User login → Dashboard → tab "Mentor"
2. Ai.tsx mounts → fetches `GET /api/schedule/upcoming`
3. Backend: find schedules where date ≥ now, isPublished=true, sorted by date asc
4. Frontend renders: "Jadwal Mentoring Mendatang" section
5. Each card: title, mentor name, date/time, "Join Zoom" button
6. Di bawahnya: mentor cards (existing)

### Scenario 3: Mentor hapus jadwal

1. Mentor lihat list jadwal di "Sesi Live" tab
2. Click icon delete pada jadwal
3. Confirm → DELETE `/api/schedule/:id`
4. Backend: verify mentorId matches → delete → success
5. Frontend: fetchSchedules() → jadwal hilang dari list

### Scenario 4: Jadwal sudah lewat

1. User buka tab "Mentor"
2. `GET /api/schedule/upcoming` → filter `date >= now`
3. Jadwal yang sudah lewat **tidak muncul** di user side
4. Di mentor side: jadwal lama masih terlihat (bisa dihapus)
