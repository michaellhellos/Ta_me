# Spec 023: Simulasi Sell Percentage Buttons

## 1. Prompt & Objective
**Prompt:** "{bantu saya kerjakan saya lagi itu misal di halaman user nya yang di bagian sumilasi yang jual nya itu bisa sual nga misal 25% 50% 75% 100% nah itu bantu saya nah bisa seperti itu }"
**Tujuan (Objective):** Menambahkan tombol pintasan persentase (25%, 50%, 75%, 100%) di panel "Jual" pada halaman Simulasi Trading. Tombol ini akan otomatis menghitung dan mengisi form "Jumlah Coin" sesuai dengan persentase dari koin yang dimiliki (owned balance).
**Kenapa tujuan ini penting:** Ini adalah fitur standard (best practice) pada semua platform bursa kripto (exchange) untuk mempermudah user melakukan order jual sebagian (partial taking profit) tanpa harus menghitung secara manual atau mengetik desimal koin yang rumit.

## 2. Analisis Codebase & Architecture
- **Kondisi Saat Ini (`frontend/src/pages/Simulasi.tsx`):**
  Pada Spec 022, kita telah menambahkan state turunan `ownedQuantity` yang menghitung berapa banyak koin yang dimiliki saat ini berdasarkan historis `history` (dengan formula SUM BUY - SUM SELL). Kita juga telah menampilkan teks `"Tersedia: {ownedQuantity}"`.
- **Kondisi CSS (`frontend/src/pages/Simulasi.css`):**
  Panel trade saat ini menggunakan class `.trade-panel`.

## 3. Rencana Perubahan (Logic & Code)
1. **Perubahan Logic:**
   - Menambahkan deretan *button* dengan label 25%, 50%, 75%, dan 100%.
   - Tombol-tombol ini HANYA akan muncul apabila `mode === 'sell'` dan di-*render* di bawah input/form `amount`.
   - Menambahkan event `onClick` pada button persentase:
     - `25%` -> `setAmount(Number((ownedQuantity * 0.25).toFixed(6)))`
     - `50%` -> `setAmount(Number((ownedQuantity * 0.50).toFixed(6)))`
     - `75%` -> `setAmount(Number((ownedQuantity * 0.75).toFixed(6)))`
     - `100%` -> `setAmount(Number(ownedQuantity.toFixed(6)))`
   - Ini akan berintegrasi mulus dengan existing fungsi `handleTrade` yang melakukan post `/sell` sesuai isi state `amount`.

2. **Perubahan Code (File: `Simulasi.tsx`):**
   - Sisipkan blok JSX berikut di dalam form Jual sebelum tombol `Trade-btn`:
     ```tsx
     {mode === "sell" && (
       <div className="percent-buttons">
         <button onClick={() => setAmount(Number((ownedQuantity * 0.25).toFixed(6)))}>25%</button>
         <button onClick={() => setAmount(Number((ownedQuantity * 0.50).toFixed(6)))}>50%</button>
         <button onClick={() => setAmount(Number((ownedQuantity * 0.75).toFixed(6)))}>75%</button>
         <button onClick={() => setAmount(Number(ownedQuantity.toFixed(6)))}>Max</button>
       </div>
     )}
     ```

3. **Perubahan Code (File: `Simulasi.css`):**
   - Tambahkan class `.percent-buttons` dengan display grid/flex, jarak proporsional, font kecil yang elegan, background semi-transparent, list hover, active state, color grading.
     ```css
     .percent-buttons {
       display: flex;
       gap: 6px;
       margin-bottom: 20px;
     }

     .percent-buttons button {
       flex: 1;
       padding: 6px 0;
       border-radius: 6px;
       border: 1px solid var(--card-border);
       background: rgba(255, 255, 255, 0.05);
       color: var(--text-secondary);
       font-size: 12px;
       font-weight: 600;
       cursor: pointer;
       transition: all 0.2s;
     }
     .percent-buttons button:hover {
       background: rgba(239, 68, 68, 0.1);
       border-color: rgba(239, 68, 68, 0.3);
       color: #ef4444; /* warna jual merah/oranye untuk konsistensi */
     }
     ```

## 4. Pseudocode & Tracing
```text
State:
  ownedQuantity = 100 BTC
  mode = "sell"
  amount = (user input)

Action: User cliks "50%"
Trigger: setAmount(100 * 0.50) -> setAmount(50)
Result: Input field displays "50".

Action: User clicks "Sell Bitcoin"
Trigger: handleTrade() reads amount = 50. Proceeds to validate 50 <= 100. Trade passes.
```
