## 1. Tujuan yang Diinginkan

Merubah tampilan keseluruhan dari halaman `Dashboard Admin` (`DashboardAdmin.tsx` dan `DashboardAdmin.css`) agar mengikuti struktur, estetika, dan format kelas CSS dari template HTML dengan desain gelap (dark theme) yang diberikan oleh user. Desain baru ini bernama "KRIPTO-Z Executive Dashboard".

**Kenapa tujuan ini penting?**
- Desain eksisting terlihat standar. Desain baru mengusung gaya *Avant-Garde UI / Executive Control* yang lebih premium, gelap (dark mode), rapi, dengan menggunakan typography khusus (Manrope dan Inter) serta warna khusus (*surface-bright*, *primary-fixed*, dll dari konfigurasi Tailwind di head template HTML).
- Ini meningkatkan *Visual Hierarchy*, *Whitespace*, dan UX keseluruhan untuk panel administrator, sesuai dengan *System Role: Senior Frontend Architect & Avant-Garde UI Designer*.
- Minimalisme dan modernitas ditonjolkan melalui ornamen efek *glassmorphism*, gradient halus, serta tata letak sidebar dan kontainer *bento grid* yang lebih profesional.

## 3. Detail Perubahan Logika & Kode

### 3.1 Perubahan Logika
Sesuai dengan *Directive: Intentional Minimalism*, kita akan memigrasi struktur HTML yang diberikan menjadi komponen React. Karena template HTML menggunakan utilitas berbasis tipe *Tailwind CSS* (`bg-surface-container`, `text-on-surface`, `bg-gradient-to-br`), dan *Tailwind* tidak dikonfigurasi secara penuh pada global project standar, kita harus:
1. Menyalin variabel warna dan plugin dari skrip CDN `<script src="https://cdn.tailwindcss.com?...">` ke dalam ekstensi atau memetakan struktur utilitas kelas secara *native* pada `.css`. Mengingat struktur HTML sangat padat dengan kelas Tailwind, lebih baik jika kita mengadopsi styling via `DashboardAdmin.css` dengan mendefinisikan root variables untuk warna yang ada pada *tailwind.config* di template HTML, kemudian menerjemahkan beberapa container grid yang kompleks. Atau, lebih amannya untuk React, kita terjemahkan kelas utilitas HTML ke kelas spesifik (seperti `.admin-sidebar`, `.admin-main`, `.bento-card`) dan men-stylingnya dengan CSS biasa.
2. Mempertahankan state React dan fetch API eksisting (spt `stats`, `topBuyers`, `menu`) lalu merendernya dalam struktur UI baru (mengganti nilai *hardcode* di HTML dengan variabel React, seperti `128,492` menjadi `{stats.users}`).

### 3.2 Perubahan Secara Code

#### [MODIFY] `DashboardAdmin.tsx`
- **Sidebar Baru:** Merombak elemen `<div className="sidebar">` menjadi sidebar `fixed left-0 top-0 h-screen w-72...` sesuai HTML dari user (diimplementasikan via CSS classes native). Navigasi akan menggunakan *state* `menu` eksisting namun desain tombolnya menjadi gradient dengan *Material Symbols Outlined*.
- **Top Bar Baru:** Membuat `<header>` yang menampung *Search bar*, notifikasi lonceng, nama "Admin", dan foto profil.
- **Hero & Action Buttons:** Menambahkan _Welcome Message_ "Halo, Administrator 👋" dengan tombol "Unduh Laporan" dan "Buat Pengumuman".
- **Bento Grid (Cards):** Merombak total empat kartu yang ada: "Total Pengguna" (`stats.users`), "Total Mentor" (`stats.mentors`), "Koin Aktif/Aset" (`stats.coins`), dan "Pesan/Notifikasi" menggunakan style *Metric Bento Grid* dari HTML template.
- **Main Grids (Table & Content):** Mengganti `<div className="charts-grid">` eksisting menjadi `grid-cols-12` sesuai template. Karena template memiliki tabel "Update Harga & Aset" dan "Konten Edukasi Terbaru", kita mungkin menempatkan ringkasan 7 kripto teratas dan daftar edukasi singkat. Untuk chart Chart.js yang eksisting, kita akan membungkusnya dalam *glassmorphism wrapper* dan menyatukannya ke dalam layout Grid tersebut (atau menaruh tabel HTML persis seperti referensi jika mengganti chart).
- **Material Symbols:** Import Google font dan `Material Symbols Outlined` ke dalam file CSS atau `index.html`.

#### [MODIFY] `DashboardAdmin.css`
- Menulis ulang CSS untuk mendukung struktur kelas baru. Menyalin root variables warna dari JSON Tailwind yang ada di `<script id="tailwind-config">` ke `:root`.
- Menambahkan style font `Inter` dan `Manrope` serta animasi group-hover.
- Membangun styling flex dan grid sesuai `grid-cols-1 lg:grid-cols-12`, dll.

### 3.3 Kenapa Code Change Dilakukan
- Untuk menyesuaikan 100% secara *pixel-perfect* (sejauh mengandalkan CSS murni dalam React) terhadap desain HTML statis yang diinginkan user.
- Mengganti icon `lucide-react` lama dengan icon `Material Symbols Outlined` sesuai dengan HTML referensi.
- Memastikan performa layout lebih responsif dan modern.
