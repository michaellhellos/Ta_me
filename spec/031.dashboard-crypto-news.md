# Spec 031: Integrasi Berita Kripto (Crypto News) di Dashboard

## 1. Prompt & Pemahaman Tugas
**Prompt User:** "{bantu saya saya mau di halaman dasboard admin, mentor dan user nya saya mau di tambahkan berita verita tetantang crypto yang terbaru misal nya data nya di ambil dari cnn atau https://newsapi.org/v2/everything?q=bitcoin&apiKey=API_KEY}"

**Tujuan (Goal & Purpose):** 
Membawa informasi dan sentimen pasar dari dunia nyata secara *real-time* ke dalam platform Kripto-Z. Dengan mengintegrasikan API berita eksternal (menggunakan `NewsAPI` atau alternatifnya), seluruh entitas pengguna—Admin, Mentor, maupun Siswa—akan langsung disajikan headline berita fundamental mengenai Cryptocurrency tanpa perlu meninggalkan aplikasi. 

**Kenapa (Why):**
Dalam ekosistem trading, pergerakan harga tidak hanya ditentukan oleh *Technical Analysis* (chart), melainkan sangat bergantung pada *Fundamental Analysis* (berita global, regulasi, adopsi teknologi). Menampilkan berita terbaru memfasilitasi Mentor dalam memberikan analisis aktual, membantu Siswa melakukan keputusan trading virtual secara rasional, dan memberi Admin visibilitas terhadap tren makro saat ini.

## 2. Konteks Codebase Saat Ini
Berdasarkan investigasi terhadap *Frontend*, struktur halaman dashboard Kripto-Z dibedakan berdasarkan *role*, yang memiliki layout statis tersendiri:
1. **User / Siswa:** `Frontend/frontend/src/pages/Dashboard.tsx` — memiliki tab/state `menu === "beranda"` dengan seksi Portofolio dan Pasar Kripto di bagian bawah.
2. **Admin:** `Frontend/frontend/src/pages/admin/DashboardAdmin.tsx` — memiliki state `menu === "beranda"` dengan tampilan statis (Total Pengguna, Mentor, Chart Traffic, dll).
3. **Mentor:** `Frontend/frontend/src/pages/mentor/MentorDashboard.tsx` — memiliki state `menu === "live"` untuk jadwal mentoring/sesi live dan metrik siswa.

## 3. Logika Perubahan (Logical Changes)
Menginjeksi proses *fetching* di tiga halaman sekaligus akan melanggar prinsip desain *DRY (Don't Repeat Yourself)*. Oleh karena itu, logika utamanya adalah:
1.  **Arsitektur Modular:** Membangun *solitary component* bernama `CryptoNewsWidget`. Sub-sistem ini bertanggung jawab mengelola internal statenya (Loading, Data, Error) dan memanggil REST API tunggal secara asinkron (menggunakan NewsAPI endpoint `everything?q=crypto`).
2.  **Graceful Degradation:** Karena menggunakan API publik bergaya Freemium yang rentan *Rate Limit Error* (429), komponen wajib dipersenjatai lapisan pencegah ledakan UI (seperti Skeleton UI saat *loading* dan instruksi *fallback* elegan saat gagal fetch).
3.  **Injeksi Komponen Secara Invasif Minimal:** Melekatkan `<CryptoNewsWidget />` ke dalam barisan bawah *layout section* milik 3 Dashboard (Siswa, Admin, Mentor). Komponen ini secara pintar membungkus dirinya dalam kontainer responsif warisan aplikasi (*Card layout / Glassmorphism*).

## 5. Pseudocode & Struktur Data API

```typescript
// Pseudo-code CryptoNewsWidget.tsx
import React, { useState, useEffect } from "react";
import "./CryptoNewsWidget.css";

const CryptoNewsWidget = () => {
  const [news, setNews] = useState([]);
  const [loading, setLoading] = useState(true);

  useEffect(() => {
    // Memanggil API dengan query 'cryptocurrency'
    fetch("https://newsapi.org/v2/everything?q=cryptocurrency+OR+bitcoin&sortBy=publishedAt&language=en&pageSize=4&apiKey=API_KEY_ANDA")
      .then(res => res.json())
      .then(data => {
        if (data.status === "ok") setNews(data.articles);
        setLoading(false);
      })
      .catch(err => {
        console.error("Gagal mendapatkan berita:", err);
        setLoading(false);
      });
  }, []);

  if (loading) return <SkeletonNewsLayout />;

  return (
    <section className="crypto-news-container card">
      <div className="section-header">
        <h3>Berita Kripto Terkini</h3>
      </div>
      <div className="news-grid">
        {news.map(article => (
          <a href={article.url} target="_blank" className="news-card">
             <img src={article.urlToImage || 'placeholder.jpg'} alt="news" />
             <div className="news-content">
                <h4>{article.title}</h4>
                <p>{article.source.name} • {formatTimeAgo(article.publishedAt)}</p>
             </div>
          </a>
        ))}
      </div>
    </section>
  );
};

export default CryptoNewsWidget;
```

## 6. Simulasi Testcase & Tracing
**Skenario 1: API Merespon dengan Baik (200 OK)**
*   *Trace:* Pengguna membuka Admin Dashboard > Lifecycle `AdminDashboard` mounting > Merender `<CryptoNewsWidget />` > Lifecycle `CryptoNewsWidget` aktif, state `loading: true` > Mengirim HTTP GET ke `newsapi.org`.
*   *Hasil UI:* Tampil *Skeleton shimmer* elegan selama dtk 0 hingga 0.5. Setelah payload turun, shimmer dihancurkan dan memunculkan Grid 4 Artikel bergambar.

**Skenario 2: API Terkena Rate Limit (429) atau API Key Habis**
*   *Trace:* fetch HTTP memutus error. Block `.catch` tertrigger atau `data.status === "error"`.
*   *Hasil UI:* UI tidak boleh terhenti (*crash / white screen of death*). State `loading: false`. Sistem menampilkan *Fallback text* yang estetik, cnth: "Deraian data berita sedang jeda. Berita pasar tidak tersedia secara temporal."
