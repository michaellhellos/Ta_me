## 2. Tujuan yang Dipahami

Buat halaman **Leaderboard** yang menampilkan **ranking semua user** berdasarkan **keuntungan (profit) terbesar**, di mana data leaderboard ini **di-reset setiap 1 minggu sekali**.

| # | Fitur | Detail |
|---|-------|--------|
| 1 | **Leaderboard ranking** | Tampilkan semua user (role=`user`), di-sort berdasarkan profit tertinggi |
| 2 | **Profit calculation** | Profit dihitung dari **realized gains** (transaksi SELL) dalam **minggu berjalan** |
| 3 | **Weekly reset** | Setiap **Senin 00:00 WIB**, hanya transaksi dari minggu baru yang dihitung |
| 4 | **Accessible dari Dashboard** | User bisa akses leaderboard dari tab navigasi bawah atau dari beranda |
| 5 | **Visual ranking** | Top 3 user mendapat visual khusus (🥇🥈🥉), sisanya numbered |

---

## 3. Kenapa Tujuan Ini?

### 3.1 Konteks Aplikasi

**Kripto-Z** adalah aplikasi **simulasi trading crypto** dengan uang virtual. Setiap user memulai dengan saldo Rp 1,000,000 (virtual). User bisa:
- **BUY** koin crypto (harga real-time dari CoinGecko API)
- **SELL** koin crypto
- Setiap transaksi tercatat di `Transaction` collection dengan timestamp

### 3.2 Mengapa Leaderboard?

- **Gamification** — Menambah motivasi user untuk belajar trading dengan kompetisi
- **Social proof** — User bisa lihat siapa yang paling jago trading
- **Weekly reset** — Mendorong konsistensi: tidak cukup untung sekali, harus konsisten tiap minggu

### 3.3 Mengapa Weekly Bukan All-Time?

- All-time leaderboard cenderung **stale** — user lama selalu di atas
- Weekly memberikan **kesempatan equal** untuk semua user setiap minggu baru
- Lebih **engaging** — user yang baru join tidak langsung discouraged

### 3.4 Existing Endpoint yang Relevan

Backend sudah punya `GET /api/trade/mentor/students` (`routes/trade.js` line 271-324) yang menghitung `totalProfit` per student dari transaksi. **TAPI**:
- ❌ Tidak ada filter waktu (menghitung ALL-TIME profit)
- ❌ Hanya accessible oleh mentor (menggunakan auth middleware)
- ❌ Profit hanya dari realized gains (SELL transactions)

Untuk leaderboard, kita perlu **endpoint baru** yang:
- ✅ Filter transaksi berdasarkan **minggu berjalan** (Senin-Minggu)
- ✅ Accessible oleh **semua authenticated user**
- ✅ Menghitung profit dari SELL transactions **dalam periode minggu ini**

### 4.4 Profit Calculation Logic (Existing Pattern)

Dari `routes/trade.js` line 271-320 (`mentor/students` endpoint), profit sudah dihitung dengan pattern:

```javascript
// Existing pattern (reference):
let buyMap = {};
transactions.forEach(trx => {
  if (trx.type === "BUY") {
    if (!buyMap[trx.coinId]) {
      buyMap[trx.coinId] = { totalQty: 0, totalCost: 0 };
    }
    buyMap[trx.coinId].totalQty += trx.quantity;
    buyMap[trx.coinId].totalCost += trx.total;
  }
  if (trx.type === "SELL") {
    if (buyMap[trx.coinId]) {
      const avgBuy = buyMap[trx.coinId].totalCost / buyMap[trx.coinId].totalQty;
      const profit = (trx.price - avgBuy) * trx.quantity;
      totalProfit += profit;
    }
  }
});
```

**Penting untuk leaderboard weekly:** 
- BUY bisa terjadi kapan saja (termasuk minggu lalu)
- Tapi SELL yang dihitung profit-nya hanya yang **dalam minggu ini**
- Jadi kita tetap perlu **semua BUY** untuk menghitung avg buy price, tapi hanya SELL minggu ini yang dihitung profit-nya

#### 5.2.1 BACKEND — [MODIFY] `routes/trade.js` (New endpoint)

**Kenapa:** Butuh endpoint publik (authenticated) untuk menghitung weekly profit ranking semua user.

```
Perubahan: Tambah endpoint GET /leaderboard
─────────────────────────────────────────────────

PSEUDOCODE:

router.get("/leaderboard", auth, async (req, res) => {
  try {
    // 1. Hitung start of current week (Senin 00:00 WIB = UTC+7)
    //    WIB = UTC + 7, jadi Senin 00:00 WIB = Minggu 17:00 UTC
    const now = new Date();
    const dayOfWeek = now.getUTCDay(); // 0=Sun, 1=Mon, ...
    
    // Hitung berapa hari ke belakang untuk sampai Senin
    // Jika hari ini Senin (1) → 0 hari ke belakang
    // Jika hari ini Minggu (0) → 6 hari ke belakang
    // Jika hari ini Selasa (2) → 1 hari ke belakang, dst
    const diffToMonday = dayOfWeek === 0 ? 6 : dayOfWeek - 1;
    
    const weekStart = new Date(now);
    weekStart.setUTCDate(now.getUTCDate() - diffToMonday);
    weekStart.setUTCHours(0, 0, 0, 0);
    // Adjust untuk WIB: Senin 00:00 WIB = Minggu 17:00 UTC
    weekStart.setUTCHours(weekStart.getUTCHours() - 7);

    // 2. Ambil semua user dengan role "user"
    const users = await User.find({ role: "user" }).select("_id name");

    // 3. Untuk setiap user, hitung weekly profit
    const leaderboard = [];

    for (const user of users) {
      // Ambil SEMUA BUY transactions (tanpa filter waktu, untuk avg cost)
      const allBuys = await Transaction.find({
        userId: user._id,
        type: "BUY"
      });

      // Bangun buyMap untuk avg cost per coin
      const buyMap = {};
      allBuys.forEach(trx => {
        if (!buyMap[trx.coinId]) {
          buyMap[trx.coinId] = { totalQty: 0, totalCost: 0 };
        }
        buyMap[trx.coinId].totalQty += trx.quantity;
        buyMap[trx.coinId].totalCost += trx.total;
      });

      // Ambil SELL transactions HANYA dari minggu ini
      const weeklySells = await Transaction.find({
        userId: user._id,
        type: "SELL",
        createdAt: { $gte: weekStart }
      });

      // Hitung profit dari weekly sells
      let totalProfit = 0;
      weeklySells.forEach(trx => {
        if (buyMap[trx.coinId] && buyMap[trx.coinId].totalQty > 0) {
          const avgBuy = buyMap[trx.coinId].totalCost / buyMap[trx.coinId].totalQty;
          const profit = (trx.price - avgBuy) * trx.quantity;
          totalProfit += profit;
        }
      });

      leaderboard.push({
        userId: user._id,
        name: user.name,
        totalProfit,
        tradeCount: weeklySells.length
      });
    }

    // 4. Sort by totalProfit DESC
    leaderboard.sort((a, b) => b.totalProfit - a.totalProfit);

    // 5. Tambah rank
    const ranked = leaderboard.map((entry, index) => ({
      ...entry,
      rank: index + 1
    }));

    res.json({
      success: true,
      weekStart: weekStart.toISOString(),
      data: ranked
    });

  } catch (err) {
    console.error(err);
    res.status(500).json({ message: "Server error" });
  }
});

─────────────────────────────────────────────────
```

> **Note tentang "Weekly Reset":**
> TIDAK perlu cron job atau scheduled task. "Reset" dilakukan secara **implicit** — endpoint menghitung profit berdasarkan `createdAt >= weekStart` (Senin 00:00 WIB). Saat minggu baru dimulai, `weekStart` otomatis bergeser, sehingga hanya transaksi minggu baru yang dihitung. Ini lebih simple, reliable, dan stateless.

## 10. Catatan Implementasi

### Tidak Perlu Dependencies Baru
- Semua library yang dibutuhkan sudah installed
- `lucide-react` sudah punya `Trophy` icon

### Tidak Perlu Cron Job
- Weekly reset dilakukan secara **query-based** (filter `createdAt >= weekStart`)
- Tidak ada data yang dihapus atau di-reset secara destructive
- Setiap kali endpoint dipanggil, `weekStart` dihitung ulang berdasarkan tanggal saat ini

### Backward Compatibility
- Endpoint baru (`/leaderboard`) tidak mengubah endpoint yang sudah ada
- Dashboard navigation tetap backward compatible (tab baru ditambahkan)
- Tidak ada schema change ke model yang sudah ada

### Performance Consideration
- Untuk skala kecil (< 100 user), loop per-user acceptable
- Jika scale: bisa optimasi dengan MongoDB aggregation pipeline
- Cache TTL 60 detik bisa ditambahkan jika perlu

### Timezone
- Week start dihitung berdasarkan **WIB (UTC+7)** karena target audience Indonesia
- Backend menghitung offset: Senin 00:00 WIB = Minggu 17:00 UTC

## 11. Pertanyaan untuk Review

> [!IMPORTANT]
> **Pertanyaan sebelum implementasi:**

1. **Tab ke-6 atau replace?** Bottom nav saat ini punya 5 tab. Menambah tab ke-6 (`leaderboard`) mungkin terlalu banyak di mobile. Alternatif:
   - **A:** Tambah sebagai tab ke-6 (biar semua fitur accessible)
   - **B:** Leaderboard jadi section di beranda (scroll ke bawah)
   - **C:** Ganti salah satu tab (misalnya merge dengan fitur lain)

2. **User tanpa SELL trade minggu ini tetap muncul?** Spec saat ini menampilkan SEMUA user (termasuk yang profit $0). Alternatif: hanya tampilkan user yang sudah trading minggu ini.

3. **Data yang ditampilkan cukup?** Saat ini: rank, nama, profit $, jumlah trade. Apakah perlu tambahan (e.g., profit %, coin yang ditradingkan)?
