## 2. Tujuan yang Dipahami

Berdasarkan prompt, terdapat kebutuhan untuk menambahkan fitur **"Redeem XP" (Tukar XP dengan Hadiah)** pada halaman Belajar.

Tujuan utama:
1. **Membuat XP Dapat Di-klik:** Bagian yang menampilkan nilai XP (khususnya *Total XP Banner*) dibuat interaktif. Ketika diklik, akan memunculkan sebuah popup (modal) penukaran hadiah.
2. **Sistem Konversi:** Menyediakan opsi penukaran XP ke saldo uang (virtual balance). 
   - 10 XP = $100
   - 100 XP = $1000
3. **Proses Redeem:** Jika XP user mencukupi, user bisa menebus hadiah tersebut. Sistem kemudian akan memotong XP yang "Available" dan menambahkan sejumlah saldo ke akun pengguna.

## 4. Codebase Analysis

### 4.1. Analisis Model User (`Backend/backend/models/user.js`)
- Model `User` memiliki field `balance` (default: 1000000). Saat ditukar, hadiah akan masuk ke `balance` ini.
- **Kekurangan:** Model `User` belum memiliki field pendata penukaran XP. Dibutuhkan tambahan field `redeemedXp` (Number, default 0).

### 4.2. Analisis Kontroler Nilai (`Backend/backend/controllers/nilaiController.js`)
- Rute `GET /api/nilai/user/:userId` saat ini memberikan array data nilai. Ini adalah rute yang dipanggil oleh frontend saat halaman muat.
- Harus di-modify agar return response-nya menyertakan `redeemedXp` dari dokumen `User` yang bersangkutan, contoh: `{ success: true, data: scores, redeemedXp: 20 }`.

### 4.3. Rute Baru (Redemption API)
- Harus membuat rute `POST /api/nilai/redeem` (atau di bawah user routes).
- Rute ini menerima `{ userId, amountXp }`.
- Logika rute: 
  1. Hitung total XP dari database Nilai untuk user tsb.
  2. Dapatkan objek User.
  3. Cek apakah `(totalXp - user.redeemedXp) >= amountXp`.
  4. Tambah `user.redeemedXp += amountXp`. Tambah `user.balance += rewardMoney`.
  5. Save dan return success.

### 4.4. Analisis Frontend (`Frontend/frontend/src/pages/Belajar.tsx`)
- Terdapat banner `.xp-summary`. Elemen ini sangat cocok dijadikan tombol trigger `onClick={() => setShowRedeemModal(true)}`.
- Frontend perlu men-track `redeemedXp` dari state, sehingga `availableXp = totalXp - redeemedXp`. Yang ditampilkan di banner utama adalah `availableXp`.

2. **Fungsi Baru `redeemXp`:**
```javascript
exports.redeemXp = async (req, res) => {
  try {
    const { userId, amountXp } = req.body;
    
    // Konversi nilai reward
    let reward = 0;
    if (amountXp === 10) reward = 100;
    else if (amountXp === 100) reward = 1000;
    else return res.status(400).json({ success: false, message: "Opsi redeem tidak valid" });

    // Hitung total earned XP
    const scores = await Nilai.find({ userId });
    const totalEarned = scores.reduce((sum, n) => sum + n.score, 0);

    const User = mongoose.model("User");
    const user = await User.findById(userId);
    if (!user) return res.status(404).json({ success: false, message: "User tidak ditemukan" });

    const availableXp = totalEarned - (user.redeemedXp || 0);

    if (availableXp < amountXp) {
      return res.status(400).json({ success: false, message: "XP tidak mencukupi" });
    }

    // Melakukan potongan XP dan penambahan balance
    user.redeemedXp = (user.redeemedXp || 0) + amountXp;
    user.balance = (user.balance || 0) + reward;
    await user.save();

    res.json({ 
      success: true, 
      message: `Berhasil redeem ${amountXp} XP menjadi $${reward}`,
      redeemedXp: user.redeemedXp,
      balance: user.balance
    });
  } catch (err) {
    res.status(500).json({ success: false, message: err.message });
  }
};
```

  }
};

## 7. Tracing Scenarios

### Scenario 1: User memiliki 15 Total XP, dan belum pernah Redeem
1. State saat render: `totalXp = 15`, `redeemedXp = 0`, `availableXp = 15`.
2. UI XP Summary menampilkan `15 XP Tersedia`.
3. User klik XP Summary, muncul popup.
4. Di modal, tombol `Tukar 10 XP → $100` **Aktif**.
5. Di modal, tombol `Tukar 100 XP → $1000` **Disabled**.
6. User klik tombol `Tukar 10 XP`. Request fired `POST /api/nilai/redeem` (amount: 10).
7. Backend update schema user: `redeemedXp` = 10,  `balance` = balance + 100. Rest API mengirim sukses.
8. Frontend update `redeemedXp` menjadi 10. `availableXp` sekarang = 5. Modal tertutup otomatis dan memunculkan toast sukses.
9. UI XP Summary kini menampilkan `5 XP Tersedia`.

### Scenario 2: User mengerjakan quiz untuk dapat tambahan XP
1. User dengan 5 Available XP (scenario 1) mengerjakan Quiz Baru, mendapat 10 XP.
2. Saat submit quiz sukses, local `nilaiMap` diupdate via REST POST ke `/api/nilai`. Score berhasil dicatat di backend.
3. React merender ulang: `totalEarnedXp` (Sum Array `nilaiMap`) menjadi 15 + 10 = 25 XP.
4. `redeemedXp` masih 10.
5. `availableXp` menjadi 25 - 10 = 15.
6. UI otomatis update `15 XP Tersedia (Klik untuk Tukar)`.
