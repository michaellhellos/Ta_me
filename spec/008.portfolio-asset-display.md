## 3. Kenapa Tujuan Ini?

### 3.1 BUG KRITIS: Transaction Schema Tidak Punya `symbol`

**Transaction Model** (`models/Transaction.js`):
```javascript
const transactionSchema = new mongoose.Schema({
  userId: { type: mongoose.Schema.Types.ObjectId, ref: "User", required: true },
  type: { type: String, enum: ["BUY", "SELL"] },
  coinId: String,    // ← ada
  name: String,      // ← ada
  price: Number,     // ← ada
  quantity: Number,  // ← ada
  total: Number,     // ← ada
  createdAt: { ... } // ← ada
  // ❌ TIDAK ADA: symbol
});
```

**BUY Route** (`routes/trade.js` lines 161-169):
```javascript
await Transaction.create({
  userId: user._id,
  type: "BUY",
  coinId: coin.id,   // ← "bitcoin"
  name: coin.name,   // ← "Bitcoin"
  price: coin.current_price,
  quantity: qty,
  total
  // ❌ TIDAK ADA: symbol
});
```

**Dashboard.tsx** (current, lines 128-145):
```typescript
buyTransactions.forEach((trx) => {
  const symbol = trx.symbol?.toUpperCase(); // ← trx.symbol === undefined!
  if (!symbol) return;  // ← SELALU RETURN! Semua transaksi di-skip!
  // ... grouping logic TIDAK PERNAH DIJALANKAN
});
```

**Dampak:** `grouped = {}` → `portfolioWithPrice = []` → "Kamu belum membeli kripto 🚀" **WALAUPUN user sudah punya coin**.

### 3.2 Ada Portfolio Collection yang Lebih Akurat

**Portfolio Model** (`models/Portfolio.js`) sudah punya:
```javascript
{ userId, coinId, name, symbol, price, quantity }
```

Portfolio collection di-maintain oleh BUY/SELL routes. Ini lebih robust karena:
- Sudah punya `symbol` 
- Sudah tracking quantity real-time
- Delete otomatis saat quantity ≤ 0

**Tapi:** `price` di Portfolio = **last buy price** (overwrite, bukan weighted avg). Jadi kita tetap butuh Transaction history untuk weighted average buy price.

### 3.3 Solusi: Hybrid Approach

**BEST FIX:**
1. **Tambah `symbol` ke Transaction.create** di BUY dan SELL routes (backend fix)
2. **Group portfolio by `coinId`** alih-alih `symbol` di frontend (immediate fix)
3. **Get symbol from market data** (CoinGecko response sudah punya `symbol`)

Approach #2 + #3 adalah immediate frontend fix tanpa migration. #1 adalah backend fix untuk data baru ke depan.

## 4. Codebase Analysis

### 4.1 Data Tersedia

| Source | Fields | Punya `symbol`? | Punya `coinId`? |
|--------|--------|-----------------|-----------------|
| `Transaction` (DB) | userId, type, coinId, name, price, quantity, total, createdAt | ❌ | ✅ `"bitcoin"` |
| `Portfolio` (DB) | userId, coinId, name, symbol, price, quantity | ✅ `"BTC"` | ✅ `"bitcoin"` |
| `CoinGecko API` | id, name, symbol, current_price, image, price_change_percentage_24h | ✅ `"btc"` | ✅ `"bitcoin"` |

**Key:** `coinId` tersedia di SEMUA source. `coinId` = CoinGecko `id` (e.g., `"bitcoin"`, `"ethereum"`). Group by `coinId` akan fix masalah.

### 4.2 Current Dashboard.tsx Portfolio Flow (BROKEN)

```
1. fetch /api/trade/history → Transaction[] (NO symbol)
2. filter type=BUY → buyTransactions
3. forEach: group by trx.symbol → ALWAYS undefined → skipped
4. Result: grouped = {} → portfolioWithPrice = []
5. UI: "Kamu belum membeli kripto 🚀"
```

### 4.3 Fixed Flow

```
1. fetch /api/trade/history → Transaction[] (has coinId)
2. filter type=BUY → buyTransactions
3. forEach: group by trx.coinId → "bitcoin", "ethereum", etc.
4. forEach SELL: subtract by coinId
5. For each group: match market by coinId → get symbol, image, currentPrice
6. Calculate: avgBuyPrice, totalValue, profitDollar, profitPercent
7. UI: render with all details
```

### 5.2 Frontend Fix

#### [MODIFY] `Dashboard.tsx`

**Portfolio grouping — GROUP BY `coinId` alih-alih `symbol`:**
```typescript
buyTransactions.forEach((trx) => {
  const id = trx.coinId; // ← gunakan coinId, BUKAN symbol
  if (!id) return;

  if (!grouped[id]) {
    grouped[id] = {
      coinId: id,
      name: trx.name,
      totalQuantity: 0,
      totalCost: 0,
    };
  }

  grouped[id].totalQuantity += Number(trx.quantity);
  grouped[id].totalCost += Number(trx.quantity) * Number(trx.price);
});

// Subtract sells by coinId
sellTransactions.forEach((trx) => {
  const id = trx.coinId;
  if (!id || !grouped[id]) return;
  grouped[id].totalQuantity -= Number(trx.quantity);
});
```

**portfolioWithPrice map — Get symbol + image from market data:**
```typescript
const portfolioWithPrice = Object.values(grouped)
  .filter((item: any) => item.totalQuantity > 0.000001)
  .map((item: any) => {
    const marketCoin = market.find((c) => c.id === item.coinId);
    
    const symbol = marketCoin?.symbol?.toUpperCase() || item.coinId;
    const image = marketCoin?.image || "";
    const currentPrice = Number(marketCoin?.current_price) || 0;

    // weighted avg buy price
    const totalBought = item.totalQuantity + sellTransactions
      .filter((s: any) => s.coinId === item.coinId)
      .reduce((sum: number, s: any) => sum + Number(s.quantity), 0);
    const avgBuyPrice = item.totalCost / totalBought;

    const totalValue = currentPrice * item.totalQuantity;
    const profitDollar = (currentPrice - avgBuyPrice) * item.totalQuantity;
    const profitPercent = avgBuyPrice > 0
      ? ((currentPrice - avgBuyPrice) / avgBuyPrice) * 100
      : 0;

    return {
      ...item,
      symbol,
      image,
      quantity: item.totalQuantity,
      avgBuyPrice,
      currentPrice,
      totalValue,
      profitDollar,
      profitPercent,
    };
  });
```

## 6. Tracing Scenarios

### Scenario 1: User punya 2 BUY transactions untuk Bitcoin

**Data di DB:**
```json
[
  { "type": "BUY", "coinId": "bitcoin", "name": "Bitcoin", "price": 50000, "quantity": 0.5, "total": 25000 },
  { "type": "BUY", "coinId": "bitcoin", "name": "Bitcoin", "price": 60000, "quantity": 0.5, "total": 30000 }
]
```

**Market data:** Bitcoin current_price = $55,000

**Calculation:**
1. `grouped["bitcoin"] = { totalQuantity: 1.0, totalCost: 55000 }`
2. `totalBought = 1.0` (no sells)
3. `avgBuyPrice = 55000 / 1.0 = $55,000`
4. `currentPrice = $55,000`
5. `totalValue = 55000 * 1.0 = $55,000`
6. `profitDollar = (55000 - 55000) * 1.0 = $0`
7. `profitPercent = 0.00%`

**Display:** Bitcoin | 1.000000 BTC | $55,000 | $0.00 (0.00%) | Avg Buy: $55,000 | Current: $55,000

### Scenario 2: User beli lalu jual sebagian

**Data di DB:**
```json
[
  { "type": "BUY", "coinId": "ethereum", "name": "Ethereum", "price": 3000, "quantity": 2, "total": 6000 },
  { "type": "SELL", "coinId": "ethereum", "name": "Ethereum", "price": 3500, "quantity": 1, "total": 3500 }
]
```

**Market data:** Ethereum current_price = $3,200

**Calculation:**
1. `grouped["ethereum"] = { totalQuantity: 2-1 = 1, totalCost: 6000 }`
2. `totalBought = 1 + 1(sold) = 2`
3. `avgBuyPrice = 6000 / 2 = $3,000`
4. `currentPrice = $3,200`
5. `totalValue = 3200 * 1 = $3,200`
6. `profitDollar = (3200 - 3000) * 1 = +$200`
7. `profitPercent = (200/3000) * 100 = +6.67%`

**Display:** Ethereum | 1.000000 ETH | $3,200 | +$200.00 (+6.67%) | Avg Buy: $3,000 | Current: $3,200

### Scenario 3: User jual semua coin

**Data di DB:**
```json
[
  { "type": "BUY", "coinId": "bitcoin", "name": "Bitcoin", "price": 50000, "quantity": 1, "total": 50000 },
  { "type": "SELL", "coinId": "bitcoin", "name": "Bitcoin", "price": 60000, "quantity": 1, "total": 60000 }
]
```

**Calculation:**
1. `grouped["bitcoin"] = { totalQuantity: 1-1 = 0, totalCost: 50000 }`
2. `filter: 0 > 0.000001` → **FALSE** → item removed
3. **Not displayed** (correct — user no longer owns Bitcoin)

### Scenario 4: User belum pernah trading

**Data di DB:** `[]`

**Calculation:**
1. `buyTransactions = []`, `sellTransactions = []`
2. `grouped = {}`
3. `portfolioWithPrice = []`
4. **Display:** "Kamu belum membeli kripto 🚀"
