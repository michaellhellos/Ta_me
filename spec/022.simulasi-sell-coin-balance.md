# Spec 022: Simulasi Sell Coin Balance

## 1. Prompt & Objective
**Prompt:** "{bantu saya yang di bagian user nya itu yang dihalaman simulasi nya saya mau nya itu jual nya menampilkan berapa coint yang saya punya }"
**Tujuan (Objective):** Menampilkan jumlah/saldo koin (portfolio) yang dimiliki user saat ini di halaman **Simulasi Trading**, tepatnya di bagian tab/opsi **Jual** (Sell). 
**Kenapa tujuan ini penting:** Tanpa melihat balance koin yang dimiliki, user kesulitan memperkirakan berapa banyak yang bisa mereka jual (max amount). Ini akan memperbaiki User Experience (UX) secara dramatis karena memberikan kejelasan kapasitas transaksi.

## 2. Analisis Codebase & Architecture
- **Simulasi Page (`frontend/src/pages/Simulasi.tsx`):**
  Meng-handle UI trading virtual. Saat ini memiliki state `mode` ("buy" atau "sell") dan menampilkan input `amount`.
- **Backend Data (`backend/routes/trade.js` dan model `Transaction`, `Portfolio`):**
  Terdapat endpoint `/api/trade/history` yang mengembalikan semua histori transaksi user (`BUY` dan `SELL`). Tapi tidak ada endpoint khusus `GET /portfolio` yang mengembalikan rekapnya langsung.
- **Portofolio Calculation (`frontend/src/pages/Dashboard.tsx`):**
  Di `Dashboard.tsx`, total portofolio dihitung di frontend dengan melakukan reduksi (reduce/sum) dari histori `history`. Jumlah BUY ditambahkan, jumlah SELL dikurangi berdasarkan `coinId`.

## 3. Rencana Perubahan (Logic & Code)
1. **Perubahan Logic:**
   - Karena portofolio di-derive dari `history`, komponen `Simulasi.tsx` harus melakukan fetch `/api/trade/history` kapanpun user berada di halaman tersebut (sekarang hanya di fetch saat tab dipindah ke "riwayat").
   - Setelah `history` didapat, lakukan kalkulasi `ownedQuantity` untuk `selectedCoin` saat ini. Rumusnya:
     `ownedQuantity = SUM(BUY quantity) - SUM(SELL quantity)` khusus untuk `selectedCoin.id`.
   - Di bagian UI *trade panel* saat `mode === "sell"`, tampilkan angka `ownedQuantity` ini.
   - Tambahkan fitur UX "click to max": Jika user mengklik text balance tersebut, secara otomatis form input `amount` akan terisi sebesar `ownedQuantity`.

2. **Perubahan Code (File: `Simulasi.tsx`):**
   - Hapus dependency cek `tab === "riwayat"` pada `useEffect` yang memanggil `fetchHistory`, menjadi dipanggil saat inisialisasi / mount. (Tambahkan trigger untuk me-refresh data balance setiap selesai order).
   - Buat variabel turunan (derived state) `ownedQuantity`:
     ```tsx
     const ownedQuantity = history.reduce((acc, trx) => {
       if (trx.coinId === selectedCoin?.id || trx.name === selectedCoin?.name) {
         if (trx.type === "BUY") return acc + trx.quantity;
         if (trx.type === "SELL") return acc - trx.quantity;
       }
       return acc;
     }, 0);
     ```
   - Modifikasi block render UI:
     ```tsx
     {mode === "sell" && (
       <div className="owned-balance" onClick={() => setAmount(ownedQuantity)}>
         Tersedia: {ownedQuantity.toFixed(6)} {selectedCoin.symbol.toUpperCase()}
       </div>
     )}
     ```
   - Tambahkan di `Simulasi.css` untuk kelas `.owned-balance` agarnya terlihat rapi, warna yang soft (seperti abu-abu atau hijau stat status profit), dan ada cursor pointer.

3. **Kenapa perubahan ini dilakukan?**
   - **Efisiensi:** Menggunakan endpoint `/history` terbukti efektif dalam arsitektur eksisting daripada membangun route portfolio baru di Backend.
   - **UX:** Tombol/text "Tersedia: X" yang *clickable* sudah menjadi standar industri crypto exchange (Binance, Indodax) untuk "100% Sell".

## 4. Pseudocode & Tracing
```text
State: 
  history = [{type:"BUY", coinId:"bitcoin", quantity: 2}, {type:"SELL", coinId:"bitcoin", quantity: 0.5}]
  selectedCoin = {id: "bitcoin"}

Process:
  ownedQuantity = 0
  loop trx in history:
    if trx.coinId == "bitcoin":
       if "BUY": ownedQuantity += 2 (ownedQuantity: 2)
       if "SELL": ownedQuantity -= 0.5 (ownedQuantity: 1.5)

Result: 
  ownedQuantity = 1.5

UI render:
  if mode == "sell":
     Render "Tersedia: 1.5 BTC"
```

## 5. Manual Testing Plan
1. Login sebagai User ke aplikasi.
2. Ke Dashboard -> Klik "Mulai Trading" (Ke halaman Simulasi).
3. Pilih koin yang belum pernah dibeli (misal Chainlink), klik tab "Jual".
4. Verifikasi muncul teks `Tersedia: 0.000000 LINK`.
5. Ubah ke tab "Beli", beli sejumlah 100 USD.
6. Tunggu UI merestart (success toast), History di fetch ulang.
7. Pindah tekan "Jual", pastikan teks berubah menjadi `Tersedia: [hasil kalkulasi] LINK`.
8. Klik teks "Tersedia: ...", pastikan nilai pada input otomatis berubah menjadi jumlah tersebut.
