# Spec 032: Integrasi Personal API Key dari NewsAPI

## 1. Prompt & Pemahaman Tugas
**Prompt User:** "{Your API key is: 82f66afc52af4d00a1a29427d83d79d9 ini Api key nya yang saya dapat dari newsapi}"

**Tujuan (Goal & Purpose):** 
Mengganti endpoint publik sementara (*CryptoCompare API*) dengan endpoint berbayar/developer dari **NewsAPI** menggunakan API Key resmi milik User (`82f66afc52af4d00a1a29427d83d79d9`). Pembaruan ini diperlukan agar pasokan berita lebih terpusat sesuai dengan *request* awal klien.

**Kenapa (Why):**
User telah menyediakan *API Key* probadi. Meskipun URL pada *fetch* sudah disisipkan secara manual oleh klien ke dalam komponen ekosistem front-end (`CryptoNewsWidget.tsx`), skema respon data spesifik (JSON struktur) antara CryptoCompare dan NewsAPI jelas berbeda dramatis. Jika kita hanya mengganti URL tanpa merubah *parser* datanya, antarmuka akan pecah/error (karena mencari field `data.Data` yang tidak ada pada `NewsAPI`). Oleh karena itu, *mapping* properti harus dibongkar dan disesuaikan.

## 2. Konteks Codebase Saat Ini
*   **File:** `Frontend/frontend/src/components/CryptoNewsWidget.tsx`
*   **Kondisi Sekarang:** Klien sudah mengganti titik *fetch* ke `https://newsapi.org/v2/everything...`. Namun, *interface* TypeScript `NewsArticle` dan mekanisme `if (data.Message === "News list successfully returned")` masih tertahan pada logika rute lama.

## 3. Logika Perubahan (Logical Changes)
1.  **Transformasi Interface TypeScript:** Menyesuaikan definisi `NewsArticle` agar selaras dengan skema respon NewsAPI:
    *   Dari `source_info.name` menjadi `source.name`
    *   Dari `imageurl` menjadi `urlToImage`
    *   Dari `published_on` (Epoch Unix timestamp) menjadi `publishedAt` (ISO-8601 String).
2.  **Transformasi Fungsi Fetch:**
    *   Pengecekan validasi respon diubah menjadi `if (data.status === "ok")`.
    *   Memasukkan array berita dari `data.articles`, bukan `data.Data`.
3.  **Transformasi Fungsi Rentang Waktu:** `formatTimeAgo` dimodifikasi. Karena sekarang *input*-nya adalah *ISO String UTC*, maka *string* tersebut harus dikonversi ke format *Unix timestamp* internal sebelum diverifikasi terhadap waktu sekarang.
