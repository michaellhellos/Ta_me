## 1. Prompt / Task


---

## 2. Tujuan yang Dipahami

Menambahkan **fitur komentar (comment)** pada setiap post di forum komunitas, sehingga:

| # | Capability | Keterangan |
|---|-----------|------------|
| 1 | **User bisa comment** | Semua role (user, mentor, admin) bisa mengirim komentar pada post |
| 2 | **Nama user ditampilkan** | Setiap komentar menampilkan nama pengirim |
| 3 | **Waktu ditampilkan** | Setiap komentar menampilkan waktu relatif (e.g. "5 menit lalu") |
| 4 | **commentsCount terupdate** | Counter `💬` pada post otomatis bertambah saat ada komentar baru |

---

## 3. Kenapa Tujuan Ini?

1. **Forum saat ini one-way** — Mentor broadcast, user hanya bisa like. Tidak ada interaksi diskusi. Ini membuat forum kurang engaging.
2. **`commentsCount` sudah ada di model** — `CommunityPost.js` sudah punya field `commentsCount` (default: 0) tapi tidak pernah diupdate karena belum ada fitur komentar.
3. **`💬` di frontend sudah dirender** — `Komunitas.tsx` sudah menampilkan `💬 {post.commentsCount}` tapi hanya sebagai teks statis, tidak bisa diklik, dan selalu 0.
4. **Auth middleware sudah meng-expose user data** — `req.user` berisi `{ _id, name, email, role }` sehingga nama commenter bisa langsung diambil.

#### 5.2.1 BACKEND — [NEW] `models/Comment.js`

**Kenapa:** Perlu model terpisah untuk komentar. Embed di CommunityPost tidak ideal karena: (1) document bisa terlalu besar jika banyak komentar, (2) query dan pagination jadi sulit.

```
Pseudocode:
─────────────────────────────────────────────────
const commentSchema = new mongoose.Schema({
  postId:     ObjectId → CommunityPost (required, indexed)
  author:     ObjectId → User (required)
  authorName: String (required)       // ← denormalized for fast display
  role:       String enum ["user", "mentor", "admin"]
  text:       String (required, trimmed, maxlength: 1000)
}, { timestamps: true })

// Index untuk query cepat: semua komentar untuk 1 post
commentSchema.index({ postId: 1, createdAt: 1 })

module.exports = mongoose.model("Comment", commentSchema)
─────────────────────────────────────────────────
```

**Kenapa `authorName` denormalized?**
- Setiap komentar perlu tampilkan nama.
- Jika tidak denormalized, harus `.populate("author", "name")` setiap query — lebih lambat.
- Trade-off: jika user ganti nama, nama di comment lama tidak update. Acceptable untuk app ini.

**Kenapa `text` plain text bukan HTML?**
- Comment tidak butuh WYSIWYG — kompleksitas tidak sebanding dengan value
- Plain text lebih aman (tidak perlu sanitasi HTML)
- Task hanya minta "bisa komen" — tidak minta rich text di komen

#### 5.2.2 BACKEND — [MODIFY] `routes/community.js`

**Kenapa:** Tambah 2 endpoint baru untuk create dan get comments. Juga update `commentsCount` di post saat comment dibuat.

```
Pseudocode — POST /api/community/:id/comment (NEW):
─────────────────────────────────────────────────
1. Middleware: auth (semua role bisa comment, BUKAN mentorOnly)
2. Receive body: { text }
3. VALIDATE:
   - postId valid ObjectId?
   - Post exists and isActive?
   - text not empty?
   - text.length <= 1000?
4. CREATE comment:
   - postId: req.params.id
   - author: req.user._id
   - authorName: req.user.name
   - role: req.user.role
   - text: text.trim()
5. INCREMENT commentsCount on CommunityPost:
   - CommunityPost.findByIdAndUpdate(postId, { $inc: { commentsCount: 1 } })
6. Return created comment with author info
─────────────────────────────────────────────────

Pseudocode — GET /api/community/:id/comments (NEW):
─────────────────────────────────────────────────
1. Middleware: auth
2. VALIDATE: postId valid?
3. QUERY: Comment.find({ postId })
   .sort({ createdAt: 1 })  // oldest first (chat-like)
4. Return array of comments
─────────────────────────────────────────────────
```

**Kenapa semua role bisa comment?**
- Task bilang "user bisa melakukan komen" — jadi user biasa harus bisa.
- Mentor dan admin juga harus bisa reply diskusi.
- Beda dengan CREATE POST yang mentor-only.

**Kenapa `$inc` bukan re-count?**
- `$inc: { commentsCount: 1 }` = atomic increment, sangat cepat
- Re-count (`Comment.countDocuments({ postId })`) lebih akurat tapi lebih lambat
- Untuk app ini, atomic increment cukup reliable

### 7.1 Comment Model Schema

```javascript

    type: String,
    required: true,
    trim: true,
    maxlength: 1000
  }
}
// + timestamps: true → createdAt, updatedAt auto-generated
```

### 7.2 Create Comment Route (Pseudocode)

```
FUNCTION createComment(req, res):
  INPUT: req.params.id (postId), req.body.text, req.user
  
  1. IF text is empty OR text.trim() === ""
       → RETURN 400 "Komentar tidak boleh kosong"
  
  2. IF text.length > 1000
       → RETURN 400 "Komentar maksimal 1000 karakter"
  
  3. post = CommunityPost.findById(postId)
     IF !post OR !post.isActive
       → RETURN 404 "Post tidak ditemukan"
  
  4. comment = new Comment({
       postId: postId,
       author: req.user._id,
       authorName: req.user.name,
       role: req.user.role,
       text: text.trim()
     })
     comment.save()
  
  5. CommunityPost.findByIdAndUpdate(postId, { $inc: { commentsCount: 1 } })
  
  6. RETURN 201 { success: true, comment }
```

### 7.3 Get Comments Route (Pseudocode)

```
FUNCTION getComments(req, res):
  INPUT: req.params.id (postId)
  
  1. comments = Comment.find({ postId: postId })
       .sort({ createdAt: 1 })  // oldest first
  
  2. RETURN { success: true, data: comments }
```

### 7.4 Frontend Toggle + Fetch (Pseudocode)

```
STATE:
  expandedComments: Record<string, boolean> = {}
  commentsData: Record<string, Comment[]> = {}
  commentInputs: Record<string, string> = {}

FUNCTION toggleComments(postId):
  1. expandedComments[postId] = !expandedComments[postId]
  2. IF expandedComments[postId] === true AND commentsData[postId] is undefined:
       → fetchComments(postId)

FUNCTION fetchComments(postId):
  1. GET /api/community/{postId}/comments
  2. commentsData[postId] = response.data.data

FUNCTION submitComment(postId):
  1. text = commentInputs[postId]
  2. IF text is empty → return
  3. POST /api/community/{postId}/comment { text }
  4. ON SUCCESS:
     - commentsData[postId].push(response.data.comment)
     - posts = posts.map(p => p._id === postId 
         ? { ...p, commentsCount: p.commentsCount + 1 } 
         : p)
     - commentInputs[postId] = ""
```

```

    ├─ commentsData["abc123"] = [...prev, newComment]  // new comment appears at bottom

[6] UI now shows:
    [existing 3 comments]
```
