## 2. Tujuan yang Dipahami

Memperbaiki tampilan dan pengalaman pengguna pada **User Dashboard** (`/dashboard`). Dashboard ini adalah halaman utama setelah user login, berisi 5 tab: Beranda, Simulasi, Belajar, Komunitas, dan Ai (Mentor).

Fokus utama:
- **Beranda tab** — tampilan pertama yang dilihat user; harus informatif, menarik, dan fungsional
- **Overall UX** — navigasi, feedback, greeting, dan konsistensi visual
- **Clean code** — hapus dead code yang mengganggu maintainability

## 6. Logic / Pseudocode

### 6.1. Greeting Logic

```ts
const getGreeting = () => {
  const hour = new Date().getHours();
  if (hour < 11) return "Selamat Pagi";
  if (hour < 15) return "Selamat Siang";
  if (hour < 18) return "Selamat Sore";
  return "Selamat Malam";
};
```

### 6.2. User Name Extraction

```ts
// From /api/user/me response (already fetched):
const [userName, setUserName] = useState("");

// In fetchData:
setUserName(userData?.name || "");

// In JSX:
<h1>{getGreeting()}, {userName || "Trader"} 👋</h1>
```

### 6.3. Total P/L Calculation

```ts
// Already have: portfolioWithPrice (each has currentPrice, avgBuyPrice, quantity, profitPercent)
const totalProfitDollar = portfolioWithPrice.reduce((sum, item: any) => {
  const avgBuy = item.totalCost / item.totalQuantity;
  const pl = (item.currentPrice - avgBuy) * item.quantity;
  return sum + pl;
}, 0);

const totalProfitPercent = cryptoValue > 0
  ? ((cryptoValue - portfolioWithPrice.reduce((s, i: any) => s + i.totalCost, 0)) 
     / portfolioWithPrice.reduce((s, i: any) => s + i.totalCost, 0)) * 100
  : 0;
```

### 6.4. Logout

```ts
const handleLogout = () => {
  localStorage.clear();
  window.location.href = "/";
};
```

## 7. Tracing Scenarios

### Scenario 1: User buka Dashboard jam 09:00 WIB

1. `Dashboard` component mounts
2. `fetchData()` dipanggil — fetch coins, user, history
3. `getGreeting()` → hour=9 → returns "Selamat Pagi"
4. `userName` = userData.name = "Michaell"
5. Header renders: **"Selamat Pagi, Michaell 👋"**
6. Avatar renders: **"M"** (first letter)
7. Portfolio calculated from history → `portfolioWithPrice[]`
8. Total P/L computed → `+$2,340.50` (hijau)
9. Market rows render with `coin.image` thumbnails
10. Quick action CTA visible: "Mulai Trading →"

### Scenario 2: User klik logout

1. User klik LogOut icon di header
2. `handleLogout()` → `localStorage.clear()`
3. `window.location.href = "/"` → redirect ke Login
4. `PrivateRoute` di App.tsx → no token → redirect ke "/"

### Scenario 3: User tanpa portfolio

1. `history` = [] → `buyTransactions` = [] → `portfolioWithPrice` = []
2. `cryptoValue` = 0, `totalAsset` = balance only
3. Total P/L section → hidden (no crypto)
4. Portfolio section → empty state: "Kamu belum membeli kripto 🚀"
5. Quick action CTA prominent: "Mulai Trading →"

### Scenario 4: User portfolio rugi

1. `portfolioWithPrice` has items with negative `profitPercent`
2. `totalProfitDollar` = -$500
3. P/L badge renders: **"-$500.00"** (merah) with `-5.2%`
4. Individual portfolio items show red percentages
