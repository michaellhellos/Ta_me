## 2. Tujuan yang Dipahami

Membuat halaman **Dashboard User (tab Beranda)** di `/dashboard` menjadi **lebih rapi, lebih bagus, dan terasa premium**. Ini bukan redesign total — beranda sudah di-redesign di spec 007. Ini adalah **polish & beautification pass** untuk memperbaiki kerapihan layout, visual hierarchy, dan micro-interactions yang masih belum optimal.

### Kenapa Tujuan Ini

Dashboard beranda adalah **halaman pertama** yang user lihat setelah login. Kesan pertama sangat menentukan. Saat ini, walaupun sudah lebih baik dari versi awal (spec 007 sudah menambah greeting, P/L, coin images), masih ada beberapa masalah visual:

- Layout sections stack tanpa visual breathing room
- Market section JSX indentation berantakan (code quality)
- Duplicate CSS declarations (`.market-card` ditulis dua kali)
- Tidak ada stagger animation pada sections
- Summary stats kurang (hanya total asset, tidak ada "at a glance" info)
- Empty state portfolio terlalu plain
- Tidak ada visual separators antar sections

## 6. Logic / Pseudocode

### 6.1 Best Performer Calculation

```ts
// Hitung coin dengan profit% tertinggi dari portfolio
const bestPerformer = portfolioWithPrice.length > 0
  ? portfolioWithPrice.reduce((best: any, item: any) =>
      item.profitPercent > (best?.profitPercent || -Infinity) ? item : best
    , null)
  : null;
```

### 6.2 Stagger Animation Wrapper

```tsx
// Wrap beranda content:
{menu === "beranda" && (
  <div className="stagger-children beranda-content">
    {/* section total-asset */}
    {/* stats-row */}
    {/* quick-action-btn */}
    {/* section portofolio */}
    {/* section pasar kripto */}
  </div>
)}
```

**Catatan:** Sebelumnya beranda menggunakan React Fragment `<>...</>`. Diperlukan real DOM element (`<div>`) agar `.stagger-children` CSS bisa bekerja (karena selector `> *` butuh parent element).
