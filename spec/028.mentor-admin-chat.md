# Spec 028: Mentor-Admin Chat Feature

## 1. Prompt & Objective
**Prompt:** "{bantu saya agar mentor dapat melakukan chat dengan admin nya juga}"
**Tujuan (Objective):** Memberikan aksesibilitas komunikasi langsung antara Mentor dan Administrator. Mentor membutuhkan saluran "Support" untuk membocarakan isu sistem atau pelaporan ke Admin, sedangkan Admin membutuhkan kotak masuk (inbox) khusus untuk membalas pesan-pesan dari para Mentor.
**Kenapa tujuan ini penting:** Tanpa fitur ini, instruktur/mentor tidak memiliki sarana resolusi konflik jika terjadi bug, masalah akun, atau kebutuhan eskalasi siswa di platform.

## 2. Analisis Codebase & Architecture
- **Backend Chat Architecture (`Conversation.js`, `Message.js`, `chatRoutes.js`):** 
  Sistem chat Kripto-Z bersifat *Role-Agnostic*. Model percakapan menggunakan Array dari `User` ObjectIDs. Endpoint `POST /api/chat/conversation` hanya membutuhkan `receiverId`, terlepas apakah ia seorang siswa, mentor, ataupun admin. Oleh karena itu, **TIDAK ADA PERUBAHAN BACKEND** yang dibutuhkan untuk kapabilitas ini.
- **Identifikasi Admin:**
  Admin terdaftar di koleksi `User` dengan `email: "admin@gmail.com"` dan `role: "admin"`. Frontend bisa mendapatkan `_id` admin memanggil `GET /api/auth/user?role=admin`.
- **Frontend Mentor (`Qna.tsx`):** Tempat yang tepat untuk menaruh tombol "Hubungi Admin" adalah di Header dari Inbox Mentor (di samping judul "Inbox Chat Siswa").
- **Frontend Admin (`DashboardAdmin.tsx`):** Berbeda dengan Mentor dan User, Admin saat ini tidak memiliki komponen kotak masuk (Inbox). Kita perlu meng-create komponen baru `AdminInbox.tsx` (yang strukturnya meminjam dari struktur solid `Qna.tsx`) dan melampirkannya sebagai menu baru di Sidebar Admin.

## 3. Rencana Perubahan (Logic & Code)

### A. Perubahan Frontend Admin (`src/pages/admin/`)
1. **Buat File Baru: `AdminInbox.tsx`**
   - Komponen ini akan memuat `fetchConversations()` menggunakan endpoint `GET /api/chat/conversation`.
   - Admin akan melihat daftar obrolan yang masuk. Klik "BALAS" (Reply) merouting ke halaman percakapan `/chat/${conversationId}`.
   - Karena backend otomatis me-*filter* percakapan berdasarkan `req.user.id` (token admin), inbox ini dengan otomatis terisolasi hanya untuk pesan-pesan ke Admin.
2. **Update `DashboardAdmin.tsx`**
   - Tambahkan state menu `"inbox"`.
   - Tambahkan item di `menuItems` sidebar: `label: "Pesan Masuk"`, menggunakan icon `MessageSquare`.
   - Render `AdminInbox` jika menu `"inbox"` aktif.
3. **Styling (`DashboardAdmin.css`)**
   - Import atau tambahkan ulang styling `.qna-container`, `.qna-card`, dsb agar tampilan kotak masuk admin sama cantiknya dengan `.qna-container` glassmorphism milik mentor.

### B. Perubahan Frontend Mentor (`src/pages/mentor/Qna.tsx`)
1. Tambahkan tombol *"Hubungi Admin Support"* bewarna ungu gradiasi (Accent) di bagian `.qna-header` atau `.qna-title`.
2. Buat fungsi `handleContactAdmin()`:
   ```javascript
   // 1. Fetch _id admin
   const resAdmin = await axios.get("/api/auth/user?role=admin");
   const adminUser = resAdmin.data.data[0];
   // 2. Buat room chat
   const resConv = await axios.post("/api/chat/conversation", { receiverId: adminUser._id });
   // 3. Navigasi ke chat room
   navigate(`/chat/${resConv.data.data._id}`);
   ```

## 4. Pseudocode / Logic Tracing
```text
1. MENTOR INITIATES:
   Mentor clicks "Hubungi Admin Support".
   Frontend fetches Admin ID -> "64xx...yyy"
   Frontend POSTs /api/chat/conversation with { receiverId: "64xx...yyy" }
   Backend finds or creates the room.
   Frontend navigates to /chat/ROOM_ID.
   Mentor types: "Halo min, server error." and sends.

2. ADMIN INBOX:
   Admin logs in, clicks Menu "Pesan Masuk".
   <AdminInbox /> queries /api/chat/conversation for req.user.id (Admin).
   Backend returns ROOM_ID with lastMessage: "Halo min, server error."
   Admin clicks "BALAS".
   Admin navigates to /chat/ROOM_ID.
   Admin replies the mentor. Socket.io delivers it in real-time.
```
