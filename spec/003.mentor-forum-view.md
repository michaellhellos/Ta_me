## 3. Kenapa Tujuan Ini?

### 3.1 Mentor Tidak Bisa Melihat Post yang Sudah Dibuat

Saat ini di `MentorDashboard.tsx` baris 127-132:

```tsx
{menu === "broadcast" && (
  <div className="content-section">
    <h2>Forum Broadcast</h2>
    <div className="empty-box">Belum ada broadcast.</div>
  </div>
)}
```

Ini adalah **hardcoded placeholder** yang selalu menampilkan "Belum ada broadcast." tanpa pernah fetch data dari backend. Artinya meskipun mentor sudah membuat banyak post, mereka TIDAK bisa melihat post-post tersebut dari dashboard mereka.

### 3.2 Duplikasi Render

Di `MentorDashboard.tsx`, tab `broadcast` merender DUA hal:

1. **Baris 123:** `{menu === "broadcast" && <Createforum />}` → form create (berfungsi ✅)
2. **Baris 127-132:** Hardcoded empty box "Belum ada broadcast." → placeholder (tidak berfungsi ❌)

Kedua block ini dirender bersamaan, membuat UX membingungkan — mentor melihat form create DAN placeholder "kosong" walaupun sudah ada post.

### 3.3 User View Sudah Fungsional

`Komunitas.tsx` sudah punya forum feed yang bekerja dengan baik:
- Fetch posts via `GET /api/community/all`
- Render rich HTML content
- Like/unlike toggle
- Comment section (expand, fetch, submit)
- File download links

Tapi mentor TIDAK punya akses ke view ini karena mereka diarahkan ke `/mentor/dashboard` (bukan `/dashboard`).

### 3.4 UI Perlu Dipercantik

Tab broadcast saat ini terlihat "polos" — hanya form create + empty placeholder box. Perlu:
- Layout yang lebih structured (create form + feed dibawahnya)
- Visual distinction antara section create dan section feed
- Overall more premium look sesuai design system glassmorphism

### 5.2 Approach: Extract vs. Duplicate

Ada 2 approach untuk menampilkan forum feed di mentor page:

**Option A — Extract shared `<ForumFeed />` component:**
- Buat component shared baru yang bisa dipakai di `Komunitas.tsx` dan `MentorDashboard.tsx`
- Refactor `Komunitas.tsx` untuk pakai shared component
- **Pro:** DRY, single source of truth
- **Con:** Lebih banyak file berubah, risk regresi di user view

**Option B — Create `<MentorForumFeed />` sebagai component baru khusus mentor:**
- Buat component di `pages/mentor/MentorForumFeed.tsx`
- Logika sama seperti forum feed di `Komunitas.tsx`, tapi styling bisa disesuaikan untuk context mentor
- **Pro:** Isolated change, no risk to user view, bisa customize mentor-specific features later
- **Con:** Duplikasi code logic

**Pilihan: Option B** — Karena:
1. Mentor feed mungkin butuh fitur khusus nanti (delete own post, pin post, edit)
2. Tidak mengubah `Komunitas.tsx` → zero risk regresi
3. Styling bisa berbeda (mentor context vs user context)
4. Scope lebih kecil dan terfokus

### 5.3 Callback Pattern — Auto-refresh after create

Saat ini `Createforum.tsx` setelah submit hanya menampilkan `alert("Broadcast berhasil dikirim 🚀")`. Perlu mekanisme callback agar `MentorDashboard` bisa trigger refresh pada feed setelah post baru dibuat.

## 9. Pseudocode — MentorForumFeed Logic

### 9.1 Core Fetch Logic

```
FUNCTION fetchPosts():
  SET loading = true
  SET error = null
  TRY:
    response = GET http://localhost:5000/api/community/all
    WITH headers: { Authorization: "Bearer " + token }
    
    SET posts = response.data (array of Post)
  CATCH err:
    SET error = "Gagal memuat forum"
  FINALLY:
    SET loading = false
```

### 9.2 Like Logic

```
FUNCTION handleLike(postId):
  TRY:
    response = POST http://localhost:5000/api/community/{postId}/like
    WITH headers: { Authorization: "Bearer " + token }
    
    UPDATE posts: find post with _id === postId
      SET totalLikes = response.data.totalLikes
      IF response.data.liked:
        ADD userId to likes array
      ELSE:
        REMOVE userId from likes array
  CATCH:
    console.error
```

### 9.3 Comment Toggle + Fetch

```
FUNCTION toggleComments(postId):
  isOpening = !expandedComments[postId]
  SET expandedComments[postId] = isOpening
  
  IF isOpening AND commentsData[postId] is undefined:
    SET loadingComments[postId] = true
    TRY:
      response = GET http://localhost:5000/api/community/{postId}/comments
      SET commentsData[postId] = response.data.data
    CATCH:
      console.error
    FINALLY:
      SET loadingComments[postId] = false
```

### 9.4 Comment Submit

```
FUNCTION submitComment(postId):
  text = commentInputs[postId].trim()
  IF text is empty → return
  
  SET submittingComment[postId] = true
  TRY:
    response = POST http://localhost:5000/api/community/{postId}/comment
    WITH body: { text }
    
    APPEND response.data.comment to commentsData[postId]
    INCREMENT post.commentsCount in posts state
    CLEAR commentInputs[postId]
  CATCH:
    alert error message
  FINALLY:
    SET submittingComment[postId] = false
```

### 9.5 Auto-refresh Trigger

```
useEffect(() => {
  fetchPosts();
}, [refreshKey]);  // re-fetch when parent increments refreshKey
```

## 13. Catatan Implementasi

### Tidak Perlu Dependencies Baru
- Semua library yang dibutuhkan sudah installed (axios, react, react-router-dom)
- Styling reuse design tokens dari `global.css`
- No new npm packages required

### Import Strategy untuk CSS
`MentorForumFeed.tsx` akan import:
1. `Komunitas.css` — untuk reuse class names `.post-header`, `.post-content`, `.comment-section` dll
2. `MentorForumFeed.css` — untuk mentor-specific styling overrides

Ini menghindari duplikasi CSS rules sambil membolehkan customization.

### Backward Compatibility
- `Createforum.tsx`: `onPostCreated` prop adalah optional (`onPostCreated?: () => void`). Jika tidak diberikan, component berfungsi persis seperti sebelumnya
- User view (`Komunitas.tsx`): **ZERO changes** — tidak terpengaruh sama sekali
- Backend: **ZERO changes** — tidak terpengaruh sama sekali
- Semua existing post data tetap tampil normal
